# RadishGame 游戏开发大纲 - 定位、技术与范围

返回：[RadishGame 游戏开发大纲](cultivation-game-outline.md)

## 1. 项目定位

### 1.1 核心目标

- 项目类型：2D / 2.5D、二次元 / ACG 风格、非像素风、修仙题材长期养成游戏
- 核心体验：探索、战斗、成长、养成、经营并行推进
- 初始目标：先做单机或弱联网可玩版本，不直接做完整 MMORPG
- 平台目标：优先支持 Web、Windows、Android

### 1.2 目标产品形态

- 第一形态：单机 / 弱联网修仙 ARPG
- 第二形态：带云存档、聊天、排行榜、活动的在线版本
- 第三形态：逐步增加多人主城、组队副本、宗门、公会、交易等 MMO 要素

### 1.3 核心玩法循环

- 外出探索
- 进入战斗
- 获取资源与掉落
- 修炼、突破、强化角色
- 返回洞府经营与生产
- 再次进入更高风险区域探索

## 2. 技术路线

### 2.1 引擎与语言

- 游戏引擎：Godot 4.x
- 主要开发语言：GDScript
- 不建议首发阶段使用 Godot C#
- 原因：
  - Godot 更适合个人开发者快速进入原型开发
  - 2D 能力强，工程更轻
  - Web、Windows、Android 可统一导出
  - GDScript 学习和迭代成本更低

### 2.2 平台策略

- Web：作为试玩、传播和快速验证平台
- Windows：作为完整体验的主平台之一
- Android：作为后续扩展平台
- Flutter：不作为游戏主引擎，仅在未来需要时用于启动器、账号中心、社区或商店外壳

### 2.3 后端技术建议

- 第一阶段：本地存档，不接服务端
- 第二阶段：增加账号、云存档、排行榜、聊天等弱联网功能
- 推荐后端方案：
  - Go + WebSocket + PostgreSQL + Redis
- 可选方案：
  - Node.js + TypeScript + WebSocket + PostgreSQL + Redis

### 2.4 美术与工具链

- 引擎：Godot 4.x
- 绘制工具：Krita / Photoshop
- 2D 骨骼动画：Spine
- 地图编辑：Tiled 或 Godot TileMap
- 3D 辅助预渲染：Blender
- 文档设计：Markdown + Figma + Excel / CSV
- 版本管理：Git

## 3. 产品范围控制

### 3.1 明确不在首版完成的内容

- 真正意义上的大型 MMORPG 架构
- 开放式自由交易经济体系
- 大规模同屏玩家战斗
- 复杂 PVP 平衡
- 大量地图和海量剧情内容

### 3.2 首版必须保留的内容

- 可玩的战斗
- 明确的修仙成长线
- 至少一个洞府 / 家园玩法
- 至少一个种植或炼丹系统
- 一条可完成的主线
- 基础探索与掉落反馈

### 3.3 MVP 目标

做出一个可以完整跑通 15 到 30 分钟体验的纵向切片，而不是做一堆半成品系统。

## 4. 世界观与题材方向

### 4.1 题材基调

- 修仙世界
- 二次元 / ACG 审美
- 非像素风
- 氛围偏幻想、宗门、秘境、灵兽、灵田、法宝体系

### 4.2 世界结构建议

- 主城 / 宗门区域
- 洞府 / 家园区域
- 野外探索区域
- 副本 / 秘境区域
- 后期扩展多人公共区域

### 4.3 叙事原则

- 先确定主角成长主线
- 用主线推动大世界认知
- 用支线补足人物、门派、地区和情绪节奏
- 不做过多空泛文本堆砌，优先任务、对话、事件驱动叙事
