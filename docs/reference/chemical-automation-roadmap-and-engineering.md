# RadishGame 化工自动化游戏开发大纲 - 阶段、工程与风险

返回：[RadishGame 化工自动化游戏开发大纲](chemical-automation-game-outline.md)

## 7. 推荐开发阶段

### 7.1 第一阶段：原型验证

目标：验证基础自动化循环和流体系统是否成立。

范围：

- 一张小地图
- 三种基础资源
- 采集设施
- 基础输送系统
- 一条固体加工链
- 一条流体加工链
- 简单发电系统
- 本地存档

阶段成果：

- 玩家能从采集到生产出第一种高价值产品

### 7.2 第二阶段：核心玩法闭环

目标：让生产、扩张、风险管理真正形成循环。

范围：

- 更多单元操作设备
- 污染系统
- 事故系统
- 防御系统
- 科技树
- 更多资源中间体

阶段成果：

- 玩家需要在扩产、稳定、安全和防御之间做取舍

### 7.3 第三阶段：内容扩展

目标：增加策略深度和长线驱动力。

范围：

- 更复杂的地图环境
- 高级工艺链
- 稀有资源区域
- 多阶段敌袭
- 高级安全和环保技术

阶段成果：

- 玩家开始进行工艺改造，而不是只靠堆机器扩张

### 7.4 第四阶段：上线打磨

目标：形成完整可发布单机产品。

范围：

- UI 优化
- 教程与新手引导
- 性能优化
- 平衡调整
- 存档稳定性测试
- Web 试玩版本验证

阶段成果：

- 形成一版可发布的完整产品

## 9. 技术设计原则

### 9.1 开发原则

- 先做核心循环，再做内容扩展
- 先做单机，再考虑联网
- 先让系统跑起来，再考虑复杂画面表现
- 先做可理解的自动化，再逐步增加深度

### 9.2 工程原则

- 数据驱动优先
- 配方、设备、资源、污染、事故参数统一配置化
- 物流、流体、电力、安全分模块设计
- 优先可调试、可视化、可扩展的系统结构

### 9.3 平台原则

- 优先 Windows 平台完成开发
- Web 只作为试玩和传播补充
- 移动端在玩法与交互稳定后再评估

## 10. 风险与应对

### 10.1 最大风险

- 玩法过于硬核，劝退普通玩家
- 系统过多，开发周期失控
- 流体与物流系统性能压力过大
- 美术表现不足导致题材门槛更高

### 10.2 应对策略

- 先做清晰易懂的简化版化工逻辑
- 第一版只做少量设备和资源种类
- 用 UI 提示和图形反馈降低理解门槛
- 逐步增加复杂度，不一次堆满所有专业概念
