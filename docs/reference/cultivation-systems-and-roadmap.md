## 6. 推荐开发阶段

### 6.1 第一阶段：原型验证

目标：验证 Godot 技术路线和核心玩法循环是否成立。

范围：

- 一个主角
- 一张小地图
- 基础移动
- 基础攻击
- 两个技能
- 三种敌人
- 简单掉落
- 一个主线任务
- 一块灵田
- 本地存档

阶段成果：

- 能在 Web 或 Windows 上完成一次完整的探索、战斗、成长、回家园循环

### 6.2 第二阶段：核心玩法闭环

目标：形成修仙题材的中期可玩体验。

范围：

- 境界突破
- 功法学习与升级
- 背包和装备
- 一个 Boss
- 一个洞府建筑升级系统
- 炼丹或种植完整链路
- 更多支线任务

阶段成果：

- 玩家能清晰感受到成长驱动，并愿意重复进行探索与养成循环

### 6.3 第三阶段：弱联网版本

目标：增加在线感，而不是一开始做重 MMO。

范围：

- 账号系统
- 云存档
- 排行榜
- 邮件
- 聊天
- 世界事件广播

阶段成果：

- 游戏从单机成长为弱联网在线产品

### 6.4 第四阶段：多人化扩展

目标：逐步引入 MMORPG-like 特性。

范围：

- 多人主城同屏
- 组队副本
- 宗门系统
- 好友系统
- 交易系统
- 世界 Boss

阶段成果：

- 在不推翻原有单机核心循环的前提下，逐步增加 MMO 深度
