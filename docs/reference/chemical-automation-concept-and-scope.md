# RadishGame 化工自动化游戏开发大纲 - 定位、技术与范围

返回：[RadishGame 化工自动化游戏开发大纲](chemical-automation-game-outline.md)

## 1. 项目定位

### 1.1 核心目标

- 项目类型：2D / 2.5D、科幻工业风、自动化生产与基地经营题材游戏
- 参考方向：《异星工厂》《银河破裂者》
- 差异化方向：引入真正有化工特色的生产、流体、分离、反应、安全与污染系统
- 初始目标：先做单机可玩的自动化基地游戏，不直接做多人联机
- 平台目标：优先 Windows，其次 Web 试玩，Android 作为后续扩展

### 1.2 目标产品形态

- 第一形态：单机异星化工基地自动化游戏
- 第二形态：带基地防御、环境灾害、科技树扩展的完整版本
- 第三形态：后续可选加入协作、排行榜、创意工坊或弱联网功能

### 1.3 核心玩法循环

- 勘探与采集资源
- 建设加工设施
- 完成反应、分离、储运与供能
- 提升产率、纯度与稳定性
- 处理污染、事故与敌对威胁
- 解锁新科技并扩建基地

## 2. 技术路线

### 2.1 引擎与语言

- 游戏引擎：Godot 4.x
- 主要开发语言：GDScript
- 不建议首发阶段引入复杂多人同步
- 原因：
  - 个人开发者更适合先做完整单机系统
  - 该题材的难点在系统设计，不在联机框架
  - Godot 适合做 2D / 2.5D 单机生产模拟与基地经营

### 2.2 平台策略

- Windows：主平台，保证完整体验
- Web：后续导出试玩版，用于验证传播效果
- Android：在操作和 UI 适配成熟后再考虑

### 2.3 后端技术建议

- 第一阶段：纯本地存档
- 第二阶段：如有需要再增加云存档、排行榜、成就同步
- 推荐后端方案：
  - Go + PostgreSQL
- 可选方案：
  - Node.js + TypeScript + PostgreSQL

### 2.4 美术与工具链

- 引擎：Godot 4.x
- 场景与概念绘制：Krita / Photoshop
- 3D 辅助建模：Blender
- UI 设计：Figma
- 数据配置：Excel / CSV / JSON
- 版本管理：Git

## 3. 产品范围控制

### 3.1 明确不在首版完成的内容

- 开放世界超大地图
- 大规模 RTS 集群单位控制
- 多人联机
- 复杂剧情分支
- 过多资源种类和海量公式化系统

### 3.2 首版必须保留的内容

- 资源采集
- 电力系统
- 至少一条固体加工链
- 至少一条流体加工链
- 至少一种污染机制
- 至少一种事故机制
- 至少一种外部威胁
- 可持续扩建的基地循环

### 3.3 MVP 目标

做出一个可以完整运行 20 到 40 分钟的纵向切片，重点验证“化工自动化 + 风险管理 + 基地扩建”循环是否足够有趣。

## 4. 世界观与题材方向

### 4.1 推荐题材

- 异星化工殖民
- 末世工业基地
- 高腐蚀、高毒性、高压环境下的工业扩张

### 4.2 推荐世界结构

- 着陆基地
- 原料采集区
- 加工区
- 储运区
- 发电区
- 防御区
- 危险地貌区

### 4.3 叙事原则

- 用基地扩张推动叙事
- 用科技解锁体现世界认知升级
- 用环境异常、事故与敌袭制造中期目标
- 少做大段文本，优先事件、日志、任务、设备说明驱动叙事
