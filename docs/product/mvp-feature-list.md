# MVP Feature List

更新时间：2026-04-26

## 文档目的

这份文档用于把已经完成的“定方向”文档收束成首版 MVP 功能清单。

它不是完整游戏范围，也不是最终版本承诺。它只回答：

- 首版 MVP 必须验证哪条最小闭环。
- 哪些功能必须做，哪些可以做，哪些明确不做。
- 资源、地图、基地、战斗、成长、装备、污染、存档和 UI 的最小实现边界在哪里。
- 如何在不把联机和 MMO 复杂度前置的情况下，保留后续扩展口子。

## MVP 目标

首版 MVP 的目标不是做完整游戏，而是验证一条可反复游玩的最小闭环：

**基地准备 -> 外出探索 -> 采集 / 战斗 / 采样 -> 回基地加工 -> 产出装备 / 药剂 / 模块 -> 解锁更危险区域。**

只要这条闭环能成立，玩家就能理解 RadishCatalyst 的核心价值：

- 基地不是后台数字，而是远征后勤。
- 外出不是单纯刷怪，而是为基地带回关键资源和样本。
- 化工链不是装饰，而是把危险资源转化为通行能力和战斗能力。
- 成长不是只靠经验，而是由角色、装备、基地工艺和区域适应共同推动。

## MVP 成功标准

首版 MVP 至少应证明：

1. 玩家能从基地出发，选择一个明确探索目标。
2. 玩家能在野外完成采集、战斗、采样和撤离。
3. 玩家带回的资源能进入基地加工链。
4. 基地加工能产出至少一种直接改变下一次远征能力的物品。
5. 新产物能解锁或显著降低一个更危险区域的进入成本。
6. 存档能稳定保存世界、基地、角色、背包、区域和解锁进度。
7. UI 能让玩家看懂当前危险、资源缺口、设备状态和下一步目标。

## 必做范围

### 资源与生产

必须实现一条完整资源链：

- 至少一种基础固体资源，用于建材、零件或装备材料。
- 至少一种流体或污染相关资源，用于体现储运、处理和风险。
- 至少一种生物、污染或异常样本，用于解锁药剂、过滤模块或主线线索。
- 至少一个基地加工流程，包含输入、输出、副产物或污染影响。
- 至少一个产物能反哺远征，例如过滤模块、抗性药剂、特殊弹药、扫描模块或装备升级材料。

资源数量应少而清楚。首版重点是证明“采回来的东西能被基地转化为新能力”，不是堆大量材料名。

### 地图与区域

必须实现首颗星球的一小段可玩区域：

- 一个基地平台。
- 一个低危资源区，例如晶体矿脉或浅层矿物区。
- 一个更危险区域，例如污染边界区、受扰生态区或遗迹入口。
- 少量可采集节点、样本节点、敌人活动点和区域目标。
- 最小可建造性标记，例如可建造 / 不可建造、可破坏 / 不可破坏。
- 基础地图信息：已探索 / 未探索、资源点、危险区域和目标位置。

首版不需要完整星球，也不需要多星球。地图规模应服务闭环验证。

### 基地与设备

必须实现最小基地后勤：

- 基础库存。
- 一个或少量加工设备，例如反应器、过滤器或分离器。
- 至少一种储存设施，例如箱体或储罐。
- 一种基础地板或地基单元，用于表达建造面和基地扩建边界。
- 基础能源或运行状态。
- 设备输入、输出、处理进度和异常提示。
- 一种污染处理或污染影响的最小表达。
- 简单清理和平整规则，让玩家理解部分设备需要稳定地面。

基地不需要完整自动化工厂，但必须让玩家在出门前和回家后都做出有意义的准备与处理。

### 探索与战斗

必须实现最小外勤体验：

- 角色移动、交互、采集、采样和撤离。
- 一到两种基础敌人。
- 一种更危险的精英、区域威胁或小型首领挑战。
- 一到两种武器或工具风格。
- 基础生命、防护、能量或体力。
- 危险区域对玩家产生明确压力，例如持续污染伤害、防护消耗、视野压力或敌人强化。

战斗不需要完整 ARPG 技能体系，但必须能验证装备、补给和区域准备是否有价值。

### 角色成长

必须实现轻量成长：

- 一条基础角色进度，例如等级、外勤经验或任务解锁。
- 一套可升级防护服或装甲。
- 少量模块，例如过滤、扫描、采集或战斗增强。
- 一批基地制造的消耗品，例如治疗包、抗性药剂、特殊弹药或临时探测器。
- 一个由防护、模块或消耗品解决的区域通行门槛。

首版成长应围绕远征准备和区域推进，不做大型职业树。

### 装备与物品

必须实现最小装备闭环：

- 武器或工具。
- 防护服或装甲。
- 模块槽位或等价的装备改造方式。
- 快捷物品。
- 背包容量或负重限制。
- 基地制造与升级入口。

装备来源应以基地制造和加工产物为主，怪物和区域可以掉落样本、部件或蓝图，不应让完整随机装备掉落取代基地后勤。

### 污染与世界反馈

必须实现一种可读污染机制：

- 污染区域或污染值。
- 污染对角色、防护、敌人或区域的影响。
- 基地处理污染的方式。
- 污染资源或副产物至少有一种回收价值。

污染不应只是扣血地板。它应同时是危险、资源、区域变化和基地治理需求的入口。

### 存档与数据

必须实现本地存档：

- 离线单机可玩。
- 世界存档保存地图、区域、基地、设备、库存、污染和任务 / 解锁状态。
- 角色状态保存生命、防护、装备、背包、快捷栏和成长进度。
- 存档有版本号。
- 核心对象使用稳定 ID，而不是显示名。

首版不依赖账号、服务器、云存档或网络验证。

### UI

必须实现最小 UI：

- 冒险 HUD：生命、防护、能量或体力、武器 / 工具、快捷物品、危险提示。
- 基地 UI：库存、设备状态、输入输出、处理进度、缺料或异常原因。
- 地图 UI：已探索区域、资源点、危险区域和当前目标。
- 制造 / 加工 UI：配方输入、产出、所需设备和是否可制作。
- 存档 / 读取入口。

UI 应保持清晰克制的工业科幻现场感，但首版不需要完整组件库或高保真视觉规范。

## 可选范围

如果核心闭环已经稳定，可以考虑加入：

- 一个小型秘境或遗迹节点。
- 一个区域小首领。
- 第二种武器风格。
- 第二种防护模块。
- 一条可选副产物回收链。
- 简单任务日志。
- 简单扫描器，用于标记资源、敌人弱点或污染边界。
- 简单基地总览，用于提示缺料、污染和设备瓶颈。
- Web 试玩适配的早期技术验证。

这些内容只有在不稀释主循环验证的前提下才进入首版。

## 明确不做

首版 MVP 不做：

- 完整游戏内容。
- 多星球。
- 大地图开放世界。
- MMO 大世界。
- 联机前置依赖。
- 官方账号。
- 云存档。
- 专用服务器。
- 玩家交易、公会、排行榜或全服经济。
- 大型剧情分支。
- 完整职业体系。
- 大型天赋树。
- 大量随机词条装备。
- 赛季化终局刷装。
- 复杂蓝图自动化系统。
- 完整 Mod 平台。
- 完整玩家 Wiki 内容生产。
- 可运行的官方工具产品。

首版和首个存档 + 联机版本都不做 MMO 大世界。更远期可以保留大型多人在线演进口子，但当前阶段只按离线单机成立、后续可扩展到 2~5 人房间协作，再远期评估 2~10 人专服或高性能房间的路径设计。

## 联机边界

联机不是首版前置条件。

首版必须做到：

- 离线单机完整可玩。
- 本地存档稳定。
- 核心对象有稳定 ID。
- 世界状态和角色状态边界清楚。
- 关键交互尽量命令化，避免未来联机重构。
- 游戏规则、状态和表现层在设计上可分离。

首版不需要做到：

- 房间联机。
- 专用服务器。
- 账号系统。
- 云存档。
- 网络同步。
- 多人权限系统。

长期演进仍以 `multiplayer-and-save-architecture.md` 为准：先单机，再本地主机 / 好友房间，再专服持久化，最后才评估官方账号、云存档和更大规模在线能力。

## 首版内容建议切片

推荐首版 MVP 使用以下切片验证：

1. 基地平台：基础库存、反应 / 过滤设备、储存设施和污染处理。
2. 低危资源区：晶体矿脉、浅层矿物、基础敌人和采集节点。
3. 污染边界区：污染资源、受扰生态、持续环境压力和样本节点。
4. 基地加工链：晶体 / 矿物预处理，污染液过滤或中和，样本稳定分析。
5. 远征产物：过滤模块、抗性药剂、特殊弹药或扫描模块。
6. 区域门槛：使用远征产物进入污染边界深处或遗迹入口。
7. 小型挑战：精英敌人、污染巢穴或遗迹入口遭遇，用于验证装备和补给。

## 与后续文档的关系

后续 Godot 原型、战斗原型、数据结构、存档结构和 UI 草图都应优先服务本清单。

如果新增功能无法说明它如何强化：

**基地准备 -> 外出探索 -> 采集 / 战斗 / 采样 -> 回基地加工 -> 产出装备 / 药剂 / 模块 -> 解锁更危险区域**

就不应进入首版 MVP 必做范围。

## 当前阶段结论

RadishCatalyst 首版 MVP 的核心不是内容规模，而是闭环质量。

只要玩家能清楚体验到“基地服务冒险，冒险反哺基地”，并能通过一条资源、装备、污染和区域解锁链路进入更危险区域，首版 MVP 就完成了它的主要验证任务。
