# RadishCatalyst / 异星催化 创作开发准备总纲

## 1. 文档目的

这份文档不是单纯的灵感汇总，也不是泛泛而谈的大纲，而是面向“可以准备开始创作、开发、规划”的开工总纲。

项目正式名称：

- 英文名：`RadishCatalyst`
- 中文名：`异星催化`

它基于 `docs` 目录下已有文档中的共识与灵感，重新收束出一个更明确的方向：

- 以化工自动化为独特卖点
- 以人物探索、战斗、升级、养成为主驱动
- 以弱联网到多人协作 ARPG 为中长期方向
- 以个人开发者可执行为首要约束

它的用途是：

- 为项目立项定调
- 为后续策划文档提供上位约束
- 为技术和开发顺序定边界
- 避免项目再次滑向“大而全但无法开工”

## 2. 项目一句话定义

这不是一个单纯的工厂游戏，也不是一个传统 MMO。

它更接近：

**“以异星化工基地为核心、生存探索为外层、人物成长与战斗为主线、后期支持联机协作的 2D / 2.5D 工业科幻 ARPG。”**

更具体一点：

玩家扮演一名登陆危险星球的工程战斗员，在建设自动化化工基地的同时，外出探索、采集、战斗、升级、解锁新工艺与装备，并逐步从单人生存扩展到多人协作开发、基地共建和组队远征。

## 3. 最终方向为什么这样定

这份总纲吸收了前面几类想法：

- 化工自动化：来自你最独特的专业优势
- 人物探索与成长：来自修仙项目里最强的长期驱动力
- ARPG 战斗：保证过程手感，不让玩法变成纯后台模拟
- 联机：保留长期扩展空间，但不在首版承担全部复杂度

这个方向的价值在于：

- 它比纯修仙 MMO 更容易做出差异化
- 它比纯工厂游戏更容易提供角色代入和持续目标
- 它比纯多人游戏更适合个人开发者先做出完整可玩版本
- 它允许你把“化工专业壁垒”变成真正的核心卖点

## 4. 汲取百家之长后的核心定位

这个项目不应简单模仿某一款作品，而应明确吸收不同类型作品的长处。

### 4.1 从自动化建造类里吸收什么

- 自动化链路扩张的成就感
- 物流与流体系统带来的空间规划乐趣
- 科技树逐步解锁更复杂流程的成长感
- 从低级生产到高级工业闭环的长期目标

### 4.2 从 ARPG 里吸收什么

- 角色操控与即时战斗的手感
- 野外探索和风险回报关系
- 装备、技能、属性、Build 的持续成长
- Boss、精英怪、远征区域带来的战斗目标

### 4.3 从养成经营类里吸收什么

- 基地是“家”，不是纯数值后台
- 玩家回基地时应该有明显恢复、制造、整理、规划的节奏切换
- 生产系统不只是赚钱，而是服务于角色成长和探索推进

### 4.4 从 MMO-like 设计里吸收什么

- 在线感
- 协作目标
- 基地共建或组队探索
- 聊天、好友、公会式组织成长

但不直接照搬：

- 大规模世界同步
- 重运营经济系统
- 首发即多人主城万人同屏

## 5. 核心幻想与玩家体验

玩家应该感受到的不是“我在摆机器”，而是：

- 我在一颗危险星球上建立自己的工业前哨
- 我在亲自外出探索、战斗、回收资源、解锁禁区
- 我建设的化工设施在支撑我的装备、弹药、药剂、能源和生存能力
- 我的人物成长与基地成长是相互咬合的
- 后期我不是一个人在生存，而是在和其他玩家协作征服环境

项目需要同时满足四种体验：

- 工程感：流程、设备、效率、工艺优化
- 冒险感：外出、战斗、地图推进、未知威胁
- 成长感：角色等级、装备、技能、科技同步变强
- 归属感：基地不断扩张，逐渐成为真正的据点

## 6. 最核心的设计原则

### 6.1 基地服务于冒险，冒险反哺基地

不能把工厂系统和人物系统做成两张皮。

正确关系应该是：

- 外出探索带回稀缺资源、样本、遗物、蓝图
- 基地把这些资源转化为装备、药剂、模块、能源、建材
- 新装备和新能力让玩家能进入更危险的区域
- 更危险的区域又要求更高水平的自动化和后勤支持

### 6.2 化工自动化是特色，不是门槛

不要把玩法做成专业考试。

应该做到：

- 看得懂
- 玩得出差异
- 优化有回报
- 失误有代价

不要做到：

- 让玩家手算化工公式
- 让系统过早复杂到劝退
- 让每个设备都像真实软件仿真器

### 6.3 联机是扩展方向，不是首版依赖

首版必须能作为单机或本地伪联网版本成立。

换句话说：

- 没有多人，游戏也必须好玩
- 多人只是在后续放大资源协作、分工和远征体验

### 6.4 战斗必须是真正核心，而不是点缀

这个项目不是纯工厂游戏。

所以战斗不能只是：

- 出门打两下素材怪
- 用于拖时间的采集前置动作

而应该承担：

- 地图推进
- 区域解锁
- 资源获取门槛
- 装备成长验证
- 联机协作价值体现

## 7. 推荐的一句话题材设定

最稳的题材方向仍然是：

**异星化工殖民。**

推荐设定版本：

人类在一颗高腐蚀、高毒性、地表生态极不稳定的星球建立前哨站。玩家既是工程操作员，也是外勤战斗员。为了活下去并推进殖民计划，玩家必须建立一整套从采集、反应、分离、储运、供能到安全治理的工业体系，同时不断外出调查遗迹、清理威胁、回收技术，并最终扩展成可联机协作的大型工业据点网络。

这个设定的好处是：

- 自动化合理
- 战斗合理
- 探索合理
- 科技树合理
- 联机合理

## 8. 项目的四大支柱

### 8.1 支柱一：人物探索与即时战斗

玩法内容：

- 地图探索
- 采集
- 打怪
- 精英敌人
- Boss
- 区域解锁
- 环境 hazard 处理

设计要求：

- 操作手感清晰
- 反馈直接
- 风险与收益匹配
- 角色成长要能明显改变体验

### 8.2 支柱二：化工自动化与基地建设

玩法内容：

- 原料开采
- 固体物流
- 流体管网
- 反应与分离
- 储运缓冲
- 电力供能
- 污染处理
- 事故防护

设计要求：

- 规则可理解
- 设备辨识度高
- 优化空间明确
- 可视化反馈足够强

### 8.3 支柱三：角色升级与装备养成

玩法内容：

- 等级
- 属性成长
- 技能树
- 武器与装备
- 模块与插件
- 消耗品与药剂
- 职能定位

设计要求：

- 成长方向可选
- Build 可以形成差异
- 工业产出能直接反哺角色成长

### 8.4 支柱四：联机协作与中后期在线感

玩法内容：

- 账号
- 云存档
- 好友
- 聊天
- 组队探索
- 协作清图
- 基地共建或资源协作

设计要求：

- 联机不破坏单机核心循环
- 联机优先增加乐趣，不优先增加运维负担
- 协作玩法应建立在已有成熟玩法上，而不是另起一套系统

## 9. 核心玩法循环

项目应围绕下面这条主循环设计：

1. 在基地整理资源、补给、制造装备和消耗品
2. 外出进入野外、矿区、污染区、遗迹区探索
3. 与环境威胁和敌对生物战斗
4. 带回原料、稀有样本、蓝图、模组、任务线索
5. 用基地工艺链完成加工、提纯、合成、储运、供能扩建
6. 解锁新设备、新药剂、新武器、新区域通行能力
7. 进入更高等级区域继续探索

中期加入副循环：

- 基地污染升高 -> 敌袭增强 -> 玩家必须升级环保和防御
- 资源需求提高 -> 工艺链升级 -> 角色获得更高阶装备
- 特殊遗迹需要特定装备 / 过滤系统 / 抗性药剂 -> 基地研发对应方案

## 10. 化工特色应该如何落地

最值得保留的化工特色不是“名词很多”，而是“机制真有区别”。

### 10.1 必须保留的化工机制

- 流体比固体更重要，不只是辅助
- 中间体与副产物并存
- 部分设备有温度 / 压力 / 纯度窗口
- 工艺错误会造成效率低下而不是只有“无法合成”
- 污染处理不是装饰，而是影响战斗和地图状态
- 安全设施能预防事故，而不是只做背景图标

### 10.2 推荐保留的单元操作

首批可优先保留这些高辨识度设备：

- 反应器
- 分离器
- 储罐
- 泵
- 换热器
- 过滤器
- 吸收或洗涤装置
- 发电装置

第二批再扩展：

- 精馏塔
- 压缩机
- 离心机
- 火炬系统
- 高级废弃物回收系统

### 10.3 化工系统的正确表达方式

用这些方式表达复杂度：

- 色彩区分不同介质
- 动画表现流量和状态
- UI 直接提示“堵塞、过热、纯度不足、储罐满、压力异常”
- 让工艺优化通过产率、能耗、污染和稳定性体现

不要用这些方式制造门槛：

- 让玩家先学专业公式才能玩
- 让界面长得像工业仿真软件
- 把每个设备都做成复杂参数面板

## 11. 人物系统应该如何与工厂系统耦合

这是整个项目成败的关键点。

角色系统与基地系统必须互相咬合。

### 11.1 基地给角色提供什么

- 武器与护甲制造
- 特种弹药
- 药剂与恢复品
- 抗性装备
- 探索模块
- 环境适应装置
- 无人机与辅助装置

### 11.2 角色给基地带回什么

- 稀有矿样
- 生物样本
- 遗迹科技蓝图
- 新区域勘探数据
- 稀有催化材料
- 特殊任务奖励

### 11.3 最好的咬合方式

不要只做“打怪掉材料 -> 回家合成”。

应该加入：

- 某些工艺必须先完成遗迹调查才能解锁
- 某些区域必须先生产抗腐蚀装备才能进入
- 某些 Boss 必须先做出特定药剂或弹种才更容易打
- 某些高级设备必须通过外出回收核心部件才能建造

## 12. 联机 ARPG 应该怎么做才现实

这部分要非常克制。

### 12.1 不建议首版直接做的内容

- 大世界无缝同屏
- 复杂交易经济
- 开放 PvP
- 大规模公会战
- 高并发共享世界状态

### 12.2 建议分三步做

#### 第一步：单机可玩

- 所有系统离线可运行
- 本地存档
- 核心循环完整

#### 第二步：弱联网

- 账号系统
- 云存档
- 排行榜
- 聊天
- 邮件或活动公告

#### 第三步：协作联机

- 小队房间制组队
- 协作探索地图
- 协作 Boss
- 有限度的基地访问或共享节点

### 12.3 最稳的联机形态

比 MMO 更适合你的，是：

**“房间制在线协作 ARPG + 长线基地成长”**

这样做的好处：

- 技术复杂度明显低于 MMO
- 易于先把单机体验做扎实
- 协作能自然建立在战斗和探索上
- 服务端压力更可控

## 13. 推荐的首版产品定义

首版不要定义成“联机化工 MMO ARPG”。

首版应定义成：

**“单机可玩、可扩展为弱联网和协作联机的化工科幻 ARPG。”**

这会直接影响你后续所有技术和策划决策。

## 14. 首版必须保留的内容

### 14.1 人物侧

- 一个主角
- 基础移动
- 普攻
- 2 到 3 个技能
- 等级成长
- 基础装备系统
- 1 到 2 种 Build 方向

### 14.2 地图侧

- 一个基地区域
- 一个近郊资源区
- 一个污染危险区
- 一个遗迹或 Boss 区

### 14.3 工业侧

- 3 种基础资源
- 1 条固体加工链
- 1 条流体加工链
- 1 套电力系统
- 1 套污染系统
- 1 套事故预警或处理机制

### 14.4 目标侧

- 一条清晰的主线推进
- 一个中型里程碑目标
- 一个首版 Boss 或关卡终点

## 15. 首版明确不做的内容

- 超大地图开放世界
- 职业过多
- 海量装备词条
- 复杂拍卖行
- 多人共建持久大世界
- 复杂公会管理
- 大规模随机地图生成
- 过多资源种类
- 极度真实化工仿真

## 16. 技术与工程建议

### 16.1 客户端

- 引擎：`Godot 4.x`
- 语言：`GDScript`
- 视角：2D 或 2.5D 俯视角 / 斜视角
- 首个平台：Windows
- Web 导出作为后续试玩验证

### 16.2 服务端

首版不依赖服务端才能成立。

第二阶段建议：

- `Go + WebSocket + PostgreSQL + Redis`

如果你更偏熟悉 Web 技术栈，也可以：

- `Node.js + TypeScript + WebSocket + PostgreSQL + Redis`

### 16.3 数据与配置

建议从一开始就配置化：

- 资源
- 建筑
- 设备
- 配方
- 技能
- 装备
- 敌人
- 区域

### 16.4 目录原则

后续工程建议优先按下面几类拆：

- `client`
- `server`
- `docs`
- `assets`
- `tools`

Godot 项目内建议按功能分目录：

- `systems`
- `actors`
- `combat`
- `automation`
- `ui`
- `data`
- `maps`

## 17. 创作与策划上的第一批必定稿内容

在真正开做前，最先应该定的不是代码，而是下面这些东西。

### 17.1 世界观一句话

必须能一句话说明：

- 玩家是谁
- 为什么在这颗星球
- 为什么既要建基地又要外出战斗

### 17.2 核心玩法循环图

必须画清楚：

- 基地循环
- 外出循环
- 成长循环
- 联机扩展循环

### 17.3 首版地图结构

必须先定：

- 基地区
- 资源区
- 危险区
- Boss 区

### 17.4 首版资源链

必须先定：

- 3 种基础资源是什么
- 哪种是固体
- 哪种是流体
- 第一条核心生产链产出什么
- 它如何服务角色成长

### 17.5 首版角色成长方案

必须先定：

- 等级怎么升
- 技能怎么解锁
- 装备怎么获得
- 工厂产出如何反哺战斗

## 18. 建议的 4 周开工计划

### 第 1 周：定项目骨架

- 写世界观一句话和项目定位
- 画核心玩法循环图
- 列首版地图和资源链
- 列首版系统边界

### 第 2 周：做可玩底盘

- Godot 项目初始化
- 主角移动
- 基础攻击
- 简单敌人
- 基地场景和野外场景切换

### 第 3 周：接入工业原型

- 采集节点
- 一个基础加工设备
- 一个流体设备
- 背包和资源结算
- 回基地制造第一种成长物资

### 第 4 周：形成第一次闭环

- 一条主线任务
- 一次外出探索
- 一次回基地生产
- 一次装备或药剂升级
- 一个精英敌人或小 Boss

4 周后的目标不是“做完”，而是：

**得到一个已经证明方向成立的可玩原型。**

## 20. 最终结论

当前最值得开始做的，不是“纯工厂游戏”，也不是“纯 MMO”。

更值得立项的是：

**一个以异星化工殖民为题材、以人物探索战斗和成长为主线、以基地自动化为核心支撑、以后可逐步扩展为协作联机 ARPG 的 2D / 2.5D 工业科幻游戏。**

如果只保留一句最重要的话，那就是：

**先把“单机也成立的人物探索 + 工业基地 + 成长闭环”做出来，再去谈联网和长期运营。**
