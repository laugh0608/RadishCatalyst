# Visual And UI Direction

更新时间：2026-04-25

## 文档目的

这份文档用于在进入 MVP 功能清单和具体界面制作之前，先明确 RadishCatalyst 的美术气质和 UI 方向。

它回答以下问题：

- 游戏整体看起来应该像什么。
- UI 应该偏工程终端、ARPG HUD，还是后台管理面板。
- 冒险、基地、设备、地图、Wiki 和官方工具之间如何保持视觉一致。
- 哪些视觉方向当前不采用。

## 总体视觉结论

RadishCatalyst 的视觉方向是：

**清晰克制的工业科幻现场界面，叠加危险异星生态和化工设施的可读性。**

它不应是纯硬科幻仿真软件，也不应是厚重赛博霓虹或明亮卡通工厂。正确感受应接近：

- 玩家在一颗异常星球上操作真实可用的前哨基地。
- 设备、管线、储罐、污染和危险状态都能被快速读懂。
- UI 有工程终端感，但不会变成专业软件考试。
- 冒险画面仍然优先保证角色、敌人、弹道、危险区域和可交互物的清晰度。

## 美术气质

### 核心关键词

- 工业开拓。
- 异星生态。
- 危险现场处置。
- 化工设备可读性。
- 克制科技感。
- 明确状态反馈。

### 推荐画面方向

- 2D / 2.5D 俯视或斜俯视表达，优先保证战斗和基地布局可读。
- 基地设施应体现反应、分离、储运、供能、污染处理等工业功能，而不是泛用科幻盒子。
- 异星区域应有明显生态与地貌差异，例如腐蚀滩地、晶化矿脉、污染雾区、遗迹裂隙、活性菌毯。
- 颜色用于区分物质、危险和系统状态，不能只作为装饰。
- 设备轮廓、敌人轮廓、掉落物轮廓要优先清楚，避免复杂背景吞掉玩法信息。

### 当前不采用

- 不做纯写实硬科幻，让每个界面像真实工控软件。
- 不做霓虹赛博朋克，把画面重点变成光污染和城市夜景。
- 不做可爱休闲工厂风，避免削弱危险异星和现场处置感。
- 不做纯后台管理系统，玩家仍然要亲自探索、战斗和移动。
- 不用单一色系统治整个游戏，基地、野外、污染、遗迹和 UI 状态需要可区分。

## UI 总原则

UI 要服务两种节奏：

1. 冒险状态：玩家正在移动、战斗、探索和规避危险。
2. 基地状态：玩家正在规划、制造、排障和查看生产状态。

设计原则：

- 信息清楚优先于装饰复杂。
- 核心状态常驻，深层细节按需展开。
- 鼠标键盘优先，后续再考虑触控和手柄适配。
- 图标、颜色、标签和数字共同表达状态，不只依赖颜色。
- UI 允许有较高信息密度，但不能遮挡角色操作和地图观察。
- 同一个物品、设备、区域、怪物在游戏内、Wiki 和官方工具中使用同一套名称、图标和分类口径。

## 信息密度分层

### 第一层：即时生存信息

始终保持可见或半常驻：

- 生命。
- 护盾或防护服完整度。
- 体力 / 能量。
- 当前武器、弹药或技能资源。
- 技能冷却。
- 快捷物品。
- 当前环境危险，例如毒雾、腐蚀、辐射、污染浓度。

### 第二层：当前任务和现场提示

在玩家需要时出现：

- 当前目标。
- 附近可采集资源。
- 危险区域边界。
- 设备异常提示。
- 背包容量或关键材料不足提示。
- 主线调查线索。

### 第三层：规划与反查信息

只在面板、地图、Wiki 或官方工具中展开：

- 配方上下游。
- 设备输入输出。
- 产能、瓶颈、库存和副产物。
- 区域资源分布。
- 怪物掉落。
- 污染扩散趋势。

## 冒险 HUD

冒险 HUD 应偏 ARPG，而不是工程面板。

必须优先表达：

- 玩家是否安全。
- 当前能不能战斗。
- 还能不能继续深入。
- 周围有什么危险。
- 关键补给是否不足。

推荐结构：

- 左下或左上显示生命、防护、能量等角色状态。
- 底部显示武器、技能和快捷栏。
- 角落显示小地图、危险层级和当前目标。
- 危险状态使用边缘提示、地面覆盖层或简洁图标提示，避免大段文字弹窗。
- 进入污染、腐蚀、低氧或异常能量区域时，HUD 应快速告诉玩家危险类型和剩余安全余量。

不建议：

- 在战斗中展开复杂生产统计。
- 用大量文本说明当前所有状态。
- 把基地设备异常强行覆盖在战斗中心区域。

## 基地与设备面板

基地 UI 应偏工程终端，但保持游戏化。

设备面板需要稳定表达：

- 设备名称、等级、状态。
- 输入、输出、副产物。
- 当前库存和容量。
- 当前处理速率。
- 能源消耗。
- 关键工艺条件，例如温度、压力、纯度或稳定性。
- 异常原因，例如堵塞、缺料、满仓、过热、压力异常、污染超标。

表达方式：

- 用图标和短标签表达材料与状态。
- 用条形、环形或小型趋势图表达容量、速率和危险余量。
- 用明确颜色表达状态：正常、等待、警告、故障、停机。
- 对复杂工艺参数做游戏化抽象，不要求玩家理解真实化工公式。

基地总览需要帮助玩家发现：

- 哪条生产链缺料。
- 哪个设备成为瓶颈。
- 哪个储罐即将满。
- 哪片区域污染或事故风险升高。
- 当前基地能否支撑下一次远征。

## 地图视觉

地图不是单纯导航工具，也应服务“基地支撑冒险，冒险反哺基地”的循环。

地图至少应预留这些层级：

- 地形与区域边界。
- 已探索 / 未探索。
- 资源点。
- 危险等级。
- 污染或异常能量覆盖。
- 任务线索和调查点。
- 可回收设施或遗迹节点。
- 已建立的前哨、传送点或补给点。

地图视觉应与游戏内地貌保持一致，不要变成完全抽象的表格。资源、怪物、危险和设备图标应与 Wiki / 官方工具共用同一套分类。

## Wiki 与官方工具视觉一致性

玩家 Wiki 和官方工具不需要复刻完整游戏 HUD，但必须共享识别系统。

应保持一致的内容：

- 物品、设备、区域、怪物、配方的稳定 ID。
- 显示名称。
- 图标风格。
- 分类颜色。
- 稀有度或危险等级表达。
- `public`、`spoiler`、`internal` 的公开等级口径。
- 输入、输出、副产物、来源和用途的展示顺序。

官方工具的视觉应更偏数据查询和规划：

- 信息更密集。
- 表格、筛选、依赖图和反查链更重要。
- 不使用过多游戏内装饰。
- 仍保持工业科幻终端感，使它看起来属于同一项目。

## 可访问性与平台适配

从方向阶段就要预留：

- 字号可读，不能把关键数字压得过小。
- 颜色之外还要有图标、形状或文字状态。
- 高危提示需要有足够对比度。
- 鼠标悬停、键盘快捷键、后续触控点击区域要能共存。
- Web 试玩和 Android 远期适配不能依赖过小按钮或过密悬停信息。

## 当前阶段结论

当前不需要制作完整 UI 规范、组件库或高保真界面。

当前需要固定的是：

- 美术气质：工业科幻现场 + 异星生态。
- UI 气质：克制、高可读、工程终端感。
- 冒险 HUD：ARPG 优先，少量环境危险提示。
- 基地面板：设备状态、输入输出、瓶颈和事故风险优先。
- 地图：资源、危险、污染、探索和任务线索并重。
- Wiki / 官方工具：与游戏共享名称、图标、分类、ID 和公开等级。
