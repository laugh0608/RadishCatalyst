# Reference Positioning - 谨慎参考与差异化

返回：[Reference Positioning](reference-positioning.md)

## 可参考但要谨慎的方向

### 《Mindustry》

可以参考：

- 资源输送、生产、防御和地图推进结合得很紧。
- 敌人压力能迫使玩家优化基地布局。
- 不同地图目标可以形成短中期战役节奏。

需要避免：

- 首版变成以塔防波次和生产防线为核心。
- 简化到只有输送带、炮塔和战役关卡。
- 让角色外勤、样本、污染和装备成长被基地防御系统吞掉。

### 《Oxygen Not Included》

可以参考：

- 气体、液体、温度、压力和污染物之间的系统关系。
- 事故、堵塞、过热、缺氧等问题能通过清晰反馈被玩家理解。
- 基地不是静态建筑，而是会被环境和资源状态持续影响。

需要避免：

- 首版做成复杂生命维持和热力学模拟。
- 让玩家陷入过多维护、微操和专业系统调参。
- 把角色外勤 ARPG 体验压成基地管理模拟。

### 《Captain of Industry》

可以参考：

- 重工业链、原料处理、中间体、副产物和废弃物管理。
- 地形改造、运输、储存和工业扩张之间的关系。
- 工业生产不是只产成品，也会产出需要处理的风险和负担。

需要避免：

- 做成宏观岛屿工业管理和土方规划游戏。
- 首版引入复杂车辆、矿山、港口和大规模运输体系。
- 让化工链过早膨胀成全行业工业模拟。

### 《Techtonica》

可以参考：

- 地下或异星环境中的工厂扩张和资源发现。
- 探索新空间后逐步打开新生产链。
- 基地建设与地图开路、地形障碍之间的关联。

需要避免：

- 过度依赖第一人称洞穴探索和大型地下工厂。
- 把地形挖掘做成首版核心工作量。
- 让空间扩张替代角色战斗和化工处理。

### 《Subnautica》

可以参考：

- 环境本身带来探索压力和未知感。
- 装备、防护、载具或基地设施决定玩家能进入多深、多远、多危险的区域。
- 不同生态区通过视觉、资源、危险和通行门槛形成强差异。

需要避免：

- 做成纯生存探索或载具探索。
- 加入过重饥饿、口渴和日常生存维护。
- 让玩家主要目标变成漫游和收集，而不是基地生产反哺外勤。

### 《Valheim》

可以参考：

- 区域层级、Boss 门槛和装备推进关系清楚。
- 天气、昼夜、地形和基地位置会影响玩家准备。
- 资源和装备推动玩家进入新生态区。

需要避免：

- 做成生存沙盒建造。
- 让 Boss 和装备等级成为唯一推进方式。
- 引入过多开放世界日常劳动。

### 《Noita》

可以参考：

- 环境、液体、火、毒、爆炸等系统反应带来的惊喜。
- 不同材料和危险状态之间有可观察的连锁反应。
- 玩家能通过工具和准备改变局部环境。

需要避免：

- 做成高混沌物理沙盒。
- 首版追求大量材料实时反应。
- 让不可控连锁反应破坏基地建设和存档稳定。

### 《Terraria》 / 《Starbound》

可以参考：

- 采集、战斗、装备、区域推进和 Boss 目标的长期结构。
- 不同生态区、怪物、资源和装备之间的层级关系。
- 玩家通过准备进入更危险区域的节奏。

需要避免：

- 做成大规模 2D 沙盒挖掘建造。
- 让内容数量和装备收集压过化工基地特色。
- 首版追求海量生态、怪物和道具。

### 生存建造类

可以参考：

- 白天 / 夜晚节奏。
- 天气带来的环境压力。
- 基地作为安全区和补给点。
- 探索前准备、探索后整理的循环。

需要避免：

- 过重饥饿、口渴、体温等生存条。
- 过多日常维护。
- 天气随机惩罚玩家核心进度。

### 塔防和基地防守类

可以参考：

- 基地污染或开采行为引发敌对生态反应。
- 防御设施、警戒和事故处理。
- 某些阶段性敌袭作为压力事件。

需要避免：

- 首版变成守波次游戏。
- 玩家被迫频繁回防，打断探索和化工循环。
- 防御塔系统规模超过角色战斗和基地生产本身。

### 轻量开放世界 ARPG

可以参考：

- 区域危险分层。
- 装备与通行能力绑定。
- 精英敌人、区域事件和秘境入口。
- 探索、战斗、回收、升级的节奏。

需要避免：

- 大量刷装词条成为唯一目标。
- 地图内容铺太大但核心生产链不成立。
- 任务和剧情与化工基地脱节。

## 时间与天气参考口径

时间和天气系统值得保留，但应服务玩法，而不是只做氛围。

可以吸收的方向：

- 白天 / 夜晚改变能见度、敌人活动和探索风险。
- 雨、雪、风暴、酸雾、粉尘、孢子雨等天气改变污染、能耗、设备稳定或外勤准备。
- 天气让不同星球或区域更有差异。
- 基地和装备可以通过照明、防护、过滤、排水、保温、密封等方式应对环境。

首版边界：

- 可以先做轻量昼夜或少量天气状态。
- 天气应给玩家明确提示，并允许通过装备、药剂、模块或基地设施应对。
- 不做复杂气候模拟、季节系统、全球天气网络或大规模灾害系统。

## RadishCatalyst 的差异化核心

无论参考哪些作品，本项目最重要的差异化仍是：

- 以化工基地支撑 ARPG 外勤。
- 以样本、污染、副产物和工艺解锁推动成长。
- 以装备、药剂、模块和特殊弹药把基地成果转成区域通行能力。
- 以异星生态、遗迹、污染和天气作为探索压力来源。
- 以结构化数据支撑游戏、玩家 Wiki 和官方工具长期一致。

一句话边界：

**参考作品提供节奏和体验启发，RadishCatalyst 的核心必须落在“化工生产链 + 异星外勤 ARPG + 污染与环境压力 + 基地后勤成长”上。**
