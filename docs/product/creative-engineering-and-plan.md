## 16. 技术与工程建议

### 16.1 客户端

- 引擎：`Godot 4.x`
- 语言：`GDScript`
- 视角：2D 或 2.5D 俯视角 / 斜视角
- 首个平台：Windows
- Web 导出作为后续试玩验证

### 16.2 服务端

首版不依赖服务端才能成立。

第二阶段建议：

- `Go + WebSocket + PostgreSQL + Redis`

如果你更偏熟悉 Web 技术栈，也可以：

- `Node.js + TypeScript + WebSocket + PostgreSQL + Redis`

### 16.3 数据与配置

建议从一开始就配置化：

- 资源
- 建筑
- 设备
- 配方
- 技能
- 装备
- 敌人
- 区域

### 16.4 目录原则

后续工程建议优先按下面几类拆：

- `client`
- `server`
- `docs`
- `assets`
- `tools`

Godot 项目内建议按功能分目录：

- `systems`
- `actors`
- `combat`
- `automation`
- `ui`
- `data`
- `maps`

## 17. 创作与策划上的第一批必定稿内容

在真正开做前，最先应该定的不是代码，而是下面这些东西。

### 17.1 世界观一句话

必须能一句话说明：

- 玩家是谁
- 为什么在这颗星球
- 为什么既要建基地又要外出战斗

### 17.2 核心玩法循环图

必须画清楚：

- 基地循环
- 外出循环
- 成长循环
- 联机扩展循环

### 17.3 首版地图结构

必须先定：

- 基地区
- 资源区
- 危险区
- Boss 区

### 17.4 首版资源链

必须先定：

- 3 种基础资源是什么
- 哪种是固体
- 哪种是流体
- 第一条核心生产链产出什么
- 它如何服务角色成长

### 17.5 首版角色成长方案

必须先定：

- 等级怎么升
- 技能怎么解锁
- 装备怎么获得
- 工厂产出如何反哺战斗

## 18. 建议的 4 周开工计划

### 第 1 周：定项目骨架

- 写世界观一句话和项目定位
- 画核心玩法循环图
- 列首版地图和资源链
- 列首版系统边界

### 第 2 周：做可玩底盘

- Godot 项目初始化
- 主角移动
- 基础攻击
- 简单敌人
- 基地场景和野外场景切换

### 第 3 周：接入工业原型

- 采集节点
- 一个基础加工设备
- 一个流体设备
- 背包和资源结算
- 回基地制造第一种成长物资

### 第 4 周：形成第一次闭环

- 一条主线任务
- 一次外出探索
- 一次回基地生产
- 一次装备或药剂升级
- 一个精英敌人或小 Boss

4 周后的目标不是“做完”，而是：

**得到一个已经证明方向成立的可玩原型。**

## 20. 最终结论

当前最值得开始做的，不是“纯工厂游戏”，也不是“纯 MMO”。

更值得立项的是：

**一个以异星化工殖民为题材、以人物探索战斗和成长为主线、以基地自动化为核心支撑、以后可逐步扩展为协作联机 ARPG 的 2D / 2.5D 工业科幻游戏。**

如果只保留一句最重要的话，那就是：

**先把“单机也成立的人物探索 + 工业基地 + 成长闭环”做出来，再去谈联网和长期运营。**
