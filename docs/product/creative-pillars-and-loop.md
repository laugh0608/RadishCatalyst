## 8. 项目的四大支柱

### 8.1 支柱一：人物探索与即时战斗

玩法内容：

- 地图探索
- 采集
- 打怪
- 精英敌人
- Boss
- 区域解锁
- 环境 hazard 处理

设计要求：

- 操作手感清晰
- 反馈直接
- 风险与收益匹配
- 角色成长要能明显改变体验

### 8.2 支柱二：化工自动化与基地建设

玩法内容：

- 原料开采
- 固体物流
- 流体管网
- 反应与分离
- 储运缓冲
- 电力供能
- 污染处理
- 事故防护

设计要求：

- 规则可理解
- 设备辨识度高
- 优化空间明确
- 可视化反馈足够强

### 8.3 支柱三：角色升级与装备养成

玩法内容：

- 等级
- 属性成长
- 技能树
- 武器与装备
- 模块与插件
- 消耗品与药剂
- 职能定位

设计要求：

- 成长方向可选
- Build 可以形成差异
- 工业产出能直接反哺角色成长

### 8.4 支柱四：联机协作与中后期在线感

玩法内容：

- 账号
- 云存档
- 好友
- 聊天
- 组队探索
- 协作清图
- 基地共建或资源协作

设计要求：

- 联机不破坏单机核心循环
- 联机优先增加乐趣，不优先增加运维负担
- 协作玩法应建立在已有成熟玩法上，而不是另起一套系统

## 9. 核心玩法循环

项目应围绕下面这条主循环设计：

1. 在基地整理资源、补给、制造装备和消耗品
2. 外出进入野外、矿区、污染区、遗迹区探索
3. 与环境威胁和敌对生物战斗
4. 带回原料、稀有样本、蓝图、模组、任务线索
5. 用基地工艺链完成加工、提纯、合成、储运、供能扩建
6. 解锁新设备、新药剂、新武器、新区域通行能力
7. 进入更高等级区域继续探索

中期加入副循环：

- 基地污染升高 -> 敌袭增强 -> 玩家必须升级环保和防御
- 资源需求提高 -> 工艺链升级 -> 角色获得更高阶装备
- 特殊遗迹需要特定装备 / 过滤系统 / 抗性药剂 -> 基地研发对应方案

## 10. 化工特色应该如何落地

最值得保留的化工特色不是“名词很多”，而是“机制真有区别”。

### 10.1 必须保留的化工机制

- 流体比固体更重要，不只是辅助
- 中间体与副产物并存
- 部分设备有温度 / 压力 / 纯度窗口
- 工艺错误会造成效率低下而不是只有“无法合成”
- 污染处理不是装饰，而是影响战斗和地图状态
- 安全设施能预防事故，而不是只做背景图标

### 10.2 推荐保留的单元操作

首批可优先保留这些高辨识度设备：

- 反应器
- 分离器
- 储罐
- 泵
- 换热器
- 过滤器
- 吸收或洗涤装置
- 发电装置

第二批再扩展：

- 精馏塔
- 压缩机
- 离心机
- 火炬系统
- 高级废弃物回收系统

### 10.3 化工系统的正确表达方式

用这些方式表达复杂度：

- 色彩区分不同介质
- 动画表现流量和状态
- UI 直接提示“堵塞、过热、纯度不足、储罐满、压力异常”
- 让工艺优化通过产率、能耗、污染和稳定性体现

不要用这些方式制造门槛：

- 让玩家先学专业公式才能玩
- 让界面长得像工业仿真软件
- 把每个设备都做成复杂参数面板
