- 新增基地行动二选一运行时检查：覆盖稳场补给 / 相位测绘互斥、补给分支完成态、测绘两点读数未完成兜底、两类返回收益差异，以及分支收束后不残留活跃任务。
- 启动「基地行动调度与整备系统」实现：新增集中行动调度推导层，HUD 目标 / 运行提示和五个行动相关终端统一显示行动台阶段、可选方案、补给整备收益和测绘路线提示收益。
- 完成首轮真实整备效果：稳场补给反馈会准备一次性出发补给包，相位测绘反馈会准备井系桥前线路线目标和风险预告，并在 HUD 地图中显形；后续出发需先经过行动台整备槽确认。
- 完成统一行动台第二版入口收束：稳窗回访、补给短行动和巡线短行动都由前线行动台确认；旧补给 / 巡线分台从场景移除，定义和旧目标进度保留为存档兼容。
- 接入出发前整备确认槽：补给包 / 测绘情报从 `ready` 进入 `queued` 后才会被相位回投台消耗，玩家需要在前线行动台确认整备槽，而不是让反馈自动装入下一趟出发。
- 接入基础风险收益档位：补给整备槽会记录低风险补给计划，测绘整备槽会记录信息侦测计划；行动台显示预计收益、风险和出发代价，回投消耗时记录最近执行计划。
- 接入行动队列最小版：补给 / 测绘反馈写入当前计划槽，行动台展示下一计划候选，方案终端可替换候选且不会改动当前已准备的出发槽。
- 接入压力清障分支最小版：巡线反馈后三选一新增高风险清障方案，前线只清一处压力扰点，回基地解析成防护整备收益，并复用现有出发整备槽和相位回投执行。
- 完成目标区行动提示治理：稳场补给投放点、相位测绘两点和压力清障扰点都有专用靠近提示，明确按 E 读取 / 写入 / 清理、回基地解析和前线行动台整备确认；新增自动检查覆盖三条分支目标提示和交互回执。
- 完成行动候选轮转治理：相位回投执行当前整备槽后，会记录已执行计划、清空本次出发槽，并把下一计划候选晋升为当前可确认计划；新增自动检查覆盖补给、测绘、压力清障三条轮转路径。
- 完成准备阶段通过复核：晋升后的当前计划可再次通过前线行动台确认并由相位回投执行；相位回投台靠近提示会显示本次已确认整备，避免玩家不清楚回投会装入什么。
- 完成风险收益预告治理：稳场补给、相位测绘和压力清障三条方案共享目标、收益、风险和代价预告；行动台、方案终端和相位回投台已纳入自动检查，避免不同入口口径漂移。
- 完成风险收益确认出发：玩家在行动台确认整备槽时会锁定本次出发的目标、收益、风险和代价快照；相位回投只执行已确认计划，避免多个 queued 状态被顺带消耗。

## 关键决策

- 2026-05-18 当天不新增第 4 条轻量短行动，也不提前接压力清障；先把已落地的补给 / 测绘二选一按自动检查支撑首轮通过处理。
- 自动检查优先跟真实运行时路径对齐，减少测试 helper 自己复制任务系统规则。
- 行动整备状态进入 `WorldState`，补给包和测绘情报按“反馈归档 -> 下一次出发消耗 / 载入”的路径处理，避免只靠提示文案冒充系统能力。
- “不做复杂远征系统”不应理解成“不做新系统”；行动队列、清障分支和更完整风险等级都可以进入设计，前提是服务基地服务冒险、冒险反哺基地。
- 统一行动台已经从提示口径推进到实际操作入口、出发整备槽和基础风险收益计划；下一步应转向行动队列或清障分支。
- 出发整备确认先保持轻量状态机，不引入队员、装备 loadout 或复杂成功率；后续若扩展行动队列，应继续复用这个整备槽而不是另建一次性入口。
- 风险收益先作为出发计划档位进入 `WorldState`，不引入随机失败率；后续若加入清障或队列，也应沿用同一计划槽。
- 行动队列先只保留“当前计划槽 + 下一计划候选”，不做复杂排程、队员、装备 loadout 或成功率；候选替换只改变下一计划，本趟出发执行后才晋升，不自动派发额外前线任务。
- 压力清障先只验证高风险计划口径和防护收益，不引入新战斗系统、随机成功率或长期成长货币。
