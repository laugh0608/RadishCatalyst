# Development Logs

更新时间：2026-04-25

## 目的

本目录用于维护按周分文件的开发日志，记录阶段目标、完成事项、关键决策、验证记录、风险和下一步计划。

## 命名规则

- 文件名格式：`YYYY-Www.md`
- 例如：`2026-W17.md`
- 周期按 Asia/Shanghai（UTC+8）日期记录。

## 推荐结构

每周日志建议至少包含以下部分：

- 本周目标
- 完成情况
- 关键决策
- 验证记录
- 风险与未完成项
- 下周建议

## 使用约定

- 本周内如果有多次推进，持续追加到同一周文件。
- 关键提交应在日志中记录提交号和目的。
- 如果阶段边界发生变化，除了写周志，还要同步更新对应正式文档。
- 不要求把每次小修小改都写入周志，重点记录阶段性推进和会影响后续协作的决策。
