# 2026-W19 Development Log

周期：2026-05-04 至 2026-05-10

## 本周目标

- 进行第一切片冷启动试玩审计，确认新档开始后是否能在无开发者解释的情况下完成 15 到 30 分钟闭环。
- 先处理阻断试玩的原型 UI 布局问题，再进入目标理解、设备理解、污染补救和节奏审计。

## 完成情况

- 修正原型 HUD 初始布局：存档、任务反馈和快捷栏绑定等左侧调试面板默认隐藏，避免启动后覆盖地图、玩家出生点和关键交互对象。
- 新增 `Tab` 切换调试面板显示 / 隐藏；保留右侧状态、交互提示和日志作为冷启动试玩的基础信息面板。
- 更新启动日志，提示 `Tab` 可显示或隐藏存档 / 快捷栏调试面板。
- 将 Godot 客户端运行视口从 `1600x900` 调整为 `2500x1400`，匹配当前可用试玩窗口，避免原型舞台缩在屏幕中央。
- 按新视口重新放大并居中第一切片地图，右侧 HUD 贴边布置，减少地图、提示面板和空边之间的浪费。
- 为原型交互点增加可配置标签偏移，并拉开污染区建造点、过滤器、沉积斑和遗迹入口的标签位置，降低节点名称互相遮挡。
- 修复早期保存被固定污染区敌人挡住的问题：存档校验继续要求世界 / 角色当前区域已解锁，但允许固定切片地图中的对象、敌人和建造点预先记录后续区域状态。
- 修复基础过滤模块配方解锁链：完成“带回样本”后同时解锁过滤介质配方、基础过滤模块配方和后续制造任务，避免“需要制造该模块才能解锁制造该模块配方”的自锁。
- 缩短存档槽位中不可读取状态的错误摘要，避免长错误文本覆盖调试面板；完整结果仍通过右侧日志反馈。
- 调整左侧存档调试面板行距和按钮位置，存档摘要改为短时间 / 区域信息，避免可读取槽位文字互相遮挡。
- 右侧状态面板高度加大，背包摘要过滤 0 数量条目并限制显示长度，避免关键状态被面板底部裁切。
- 建成后的地基和污染过滤器建造点改为保留已建成色块但关闭交互，污染过滤器工作点挪到两块地基中心，避免建造后表现消失或偏离地基位置。
- 三槽位原型存档面板新增“清空”按钮，可删除槽位主档和最近 3 份备份；默认槽位会同时清理旧原型存档来源，方便反复从空档开始冷启动试玩。

## 关键决策

- 当前不继续增加解释性 HUD 文本，也不提前制作正式设备面板或小地图；先恢复可试玩状态，再按冷启动审计结果决定下一步 UI 收敛方向。
