# 2026-W18 Development Log

周期：2026-04-27 至 2026-05-03

## 本周目标

- 将 Godot 客户端从空工程骨架推进到第一可玩切片的可运行原型骨架。
- 建立首批静态数据、数据加载入口和最小状态对象。
- 建立 `Boot` 到 `GameRoot` 的启动链。
- 为客户端静态数据补充可重复验证入口。

## 完成情况

- 新增首批 `client/data/` 静态数据，覆盖物品、流体、配方、建筑、装备、敌人、区域、地图对象、污染、天气和任务。
- 新增 `client/data/localization/zh_cn.json`，让首批静态数据通过本地化键读取中文显示名和描述。
- 新增 `DataRegistry`，支持加载全部静态 JSON、按稳定 ID 查询定义、读取本地化文本和输出数据表摘要。
- 新增 `WorldState`、`CharacterState`、`InventoryState` 和 `QuestState`，支持默认状态创建和 `to_dict()` / `from_dict()` 序列化。
- 新增 `Boot.tscn`、`GameRoot.tscn`、`boot.gd` 和 `game_root.gd`，将项目主场景接到 `Boot`，并在 `GameRoot` 中初始化数据和最小状态。
- 新增 `scripts/check-client-data.ps1`，校验客户端静态数据 JSON、稳定 ID、重复 ID、引用关系和中文本地化键。
- 更新 `docs/planning/current.md`，把客户端静态数据检查纳入当前默认验证。
- 新增 `Player.tscn`、`VerticalSliceMap.tscn`、`PrototypeInteractable.tscn`、`PrototypeEnemy.tscn` 和 `PrototypeHud.tscn`，形成可移动、可交互的临时切片地图。
- 新增玩家移动、原型交互对象、原型敌人、临时地图和 HUD 脚本，支持 WASD 移动、E 键交互、背包数量变化和首个任务推进。
- 新增 `scripts/check-client-scenes.ps1`，校验 Godot 场景和项目文件中的 `res://` 资源引用是否存在。
- 新增 `scripts/check-godot-client.ps1`，默认使用 `D:\Program Files\Godot\Godot_v4.6.2-stable_win64_console.exe` 做 Godot 客户端无头加载验证。
- 调整 Godot 验证脚本，将 Godot 编辑器配置、数据和缓存目录重定向到仓库内 `.godot-check-home/`，避免写入用户全局目录；当前验证使用 `--import --quit` 完成项目导入和全局类注册。
- Godot 导入验证会为 GDScript 生成 `.gd.uid` 文件，作为 Godot 4.6 脚本稳定 UID 一并保留。

## 关键决策

- 第一轮工程实现继续围绕第一可玩切片，不提前扩展完整 MVP 内容量。
- 静态数据先使用 JSON，脚本和后续工具都通过稳定 ID 引用定义。
- 原型地图先使用 `ColorRect` 和标签占位，不提前引入正式美术资产。
- 本机 Godot console 可执行文件路径固定为 `D:\Program Files\Godot\Godot_v4.6.2-stable_win64_console.exe`，用于当前阶段的无头项目加载验证。
- `--headless --path ... --quit` 和 `--check-only --script` 在当前 Windows / Godot 4.6.2 环境会触发引擎层崩溃；当前先采用稳定的 `--import --quit` 作为项目级验证。
- Godot 导入和仓库扫描不适合并行执行；当前验证命令应顺序运行，避免缓存和文件扫描互相争用。
