# Combat And Interaction Prototype - 玩家战斗与工具

返回：[Combat And Interaction Prototype](combat-and-interaction-prototype.md)

## 文档目的

这份文档用于把首版 MVP 中的外勤、战斗、地图交互和建造前置条件拆成 Godot 原型可执行边界。

它不是完整战斗设计，也不是完整建造系统方案。它只回答：

- 首版原型里玩家能做哪些动作。
- 地图上的资源、建筑、地形和敌人如何被交互。
- 哪些对象可破坏，哪些对象不可破坏。
- 建造为什么需要考虑地形高低差、平整和地基。
- 原型阶段哪些内容必须做，哪些只做预留。

## 原型目标

首版战斗与交互原型应验证：

**玩家从基地出发，在地图上移动、探索、采集、采样、战斗、清障、撤离，并把带回的资源交给基地加工。**

它的目标不是做出完整动作游戏手感，也不是做复杂地形工程，而是确认玩家在外勤过程中始终有清楚判断：

- 这里能不能走。
- 这里能不能采。
- 这里能不能破坏。
- 这里能不能建。
- 这里危险来自敌人、污染、地形，还是资源准备不足。
- 我现在应该继续深入、清障、采样，还是撤回基地。

## 玩家基础动作

原型必须支持：

- 移动。
- 面向或瞄准。
- 基础攻击。
- 使用工具。
- 采集资源。
- 采样调查。
- 拾取掉落物。
- 使用快捷物品。
- 与基地设备或地图节点交互。
- 回到基地或撤离点。

可选支持：

- 闪避或短距离冲刺。
- 简单击退。
- 简单蓄力工具动作。
- 临时扫描。

首版不要求复杂连招、空中动作、格挡反击或高精度动作取消。

## 战斗最小边界

必须实现：

- 一种通用武器或工具攻击。
- 一种近程或远程基础敌人。
- 一种更危险的精英、巢穴或小型首领挑战。
- 生命、防护和能量 / 体力中的至少两个压力来源。
- 受击反馈、死亡 / 失败反馈和撤离后保留成果的规则。
- 敌人掉落样本、部件或污染残留，而不是主要掉完整装备。

推荐首版敌人类型：

- 原生小型生物：用于验证基础移动、攻击和采集干扰。
- 受扰污染体：用于验证污染区、样本掉落和防护压力。
- 精英节点守卫：用于验证装备、药剂、特殊弹药或模块准备。

暂不做：

- 大量敌人类型。
- 复杂 Boss 阶段。
- 职业技能循环。
- 大规模弹幕。
- PvP。
- 多人同步战斗。

## 工具与武器关系

首版可以把武器和工具先做成少量可复用交互能力。

推荐：

- 工程步枪：通用攻击，验证远程战斗。
- 切割 / 冲击工具：可攻击，也可破坏部分障碍或采集矿物。
- 扫描器：标记资源、样本、危险边界或可破坏对象。
- 投放器：使用净化剂、特殊弹药或稳定剂处理污染节点。

工具不应只为战斗服务。它们应同时连接：

- 资源采集。
- 环境清障。
- 样本调查。
- 基地制造产物。
- 区域通行门槛。
