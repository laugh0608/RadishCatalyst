# Character Progression And Equipment

更新时间：2026-05-11

## 文档目的

这份文档用于定义 RadishCatalyst 的角色成长、装备、技能和 Build 在项目方向中的位置。

它不是数值表，也不是完整技能树方案。它先回答：

- 玩家为什么需要成长。
- 成长如何服务探索、战斗、基地和区域推进。
- 装备、技能、消耗品和基地生产如何咬合。
- 首版应保留哪些 RPG 深度，避免过早膨胀。

## 核心原则

角色成长必须服务主循环：

**基地生产提供装备、模块、药剂和特殊弹药，角色用这些能力进入更危险区域，带回更高价值样本和资源，再推动基地解锁更强生产能力。**

成长不能只来自刷怪经验，也不能只靠基地科技树自动变强。

玩家应该同时感受到：

- 自己的角色变强。
- 装备和工具更适配环境。
- 基地后勤让远征更稳。
- 新区域要求新的准备和 Build 取舍。

## 成长来源

### 外勤经验

来源：

- 战斗。
- 探索新地点。
- 扫描和采样。
- 完成区域目标。
- 秘境调查。

作用：

- 解锁角色基础能力。
- 提升武器、技能或外勤专精。
- 让玩家形成对某种冒险风格的投入。

外勤经验不应成为唯一成长路径。玩家只刷怪但不经营基地，不应长期推进高危区域。

### 基地工艺

来源：

- 样本分析。
- 新资源处理。
- 设备升级。
- 污染治理。
- 遗迹或秘境回收。

作用：

- 解锁装备材料。
- 解锁防护模块。
- 解锁药剂、特殊弹药和探测工具。
- 解锁区域通行条件。

基地工艺是本项目区别于传统 ARPG 的关键。它应让玩家意识到：战斗能力来自工业后勤，而不是只来自掉落装备。

### 区域适应

来源：

- 探索特定区域。
- 处理特定污染。
- 收集当地样本。
- 完成区域调查或治理。

作用：

- 获得针对环境的防护能力。
- 降低特定危险的远征成本。
- 解锁新的地图层、资源点或秘境入口。

区域适应可以作为轻量长期成长，但不应变成需要玩家在每个区域重复刷满的疲劳系统。

## 角色属性方向

首版属性应少而清晰，优先服务动作、风险和后勤判断。

推荐保留：

- 生命：承受直接伤害。
- 防护完整度：表示防护服、护盾或外勤装甲状态。
- 能量或体力：驱动闪避、工具、技能或短时防护。
- 负重或背包容量：限制远征收益和撤离判断。
- 环境抗性：对应污染、腐蚀、毒性、低温、高温或异常能量。

暂不建议做过多传统属性，例如大量力量、敏捷、智力、精神、暴击、命中、闪避和复杂抗性堆叠。首版重点应是“能不能安全进入、能不能打赢、能不能带回、值不值得继续深入”。

## 装备结构

### 武器

武器负责战斗风格。

推荐方向：

- 工程步枪或能量步枪：稳定输出，适合通用探索。
- 近程切割器或冲击工具：高风险近战，兼具破障或采集用途。
- 投射器或发射器：使用特殊弹药，适合处理群体、装甲或污染体。
- 工具型武器：兼具扫描、采集、净化或打断效果。

武器升级应消耗基地加工产物，而不是只靠掉落同名武器。

### 防护服和装甲

防护服是连接角色成长和区域通行的核心装备。

它应承担：

- 基础防御。
- 环境过滤。
- 抗腐蚀、抗毒、抗寒、抗热或抗孢子。
- 能量供应和模块插槽。
- 事故或濒危状态下的短时保护。

高危区域不应只用等级墙阻挡，也可以要求玩家具备对应防护模块、过滤耗材或稳定剂。

### 模块

模块负责 Build 取舍。

推荐模块类型：

- 防护模块：提高特定环境抗性。
- 战斗模块：改变武器、技能或弹药效果。
- 探测模块：强化扫描、资源标记或弱点识别。
- 采集模块：提高采样、破障、回收或携带效率。
- 生存模块：改善能量、治疗、撤离或背包管理。

模块不宜一开始做成海量词条。首版更适合少量明确模块，每个模块都能改变远征选择。

### 消耗品

消耗品是基地生产反哺冒险的高频接口。

推荐类型：

- 治疗和修复包。
- 抗性药剂。
- 过滤罐、净化剂或稳定剂。
- 特殊弹药。
- 临时探测器。
- 便携能源、冷却剂或应急密封剂。

消耗品应让基地准备阶段有意义，但不能让玩家每次出门都陷入繁琐补货。

## 技能方向

技能应体现前哨工程战斗员身份，而不是传统魔法职业。

推荐技能类别：

- 武器技能：射击、切割、爆破、击退或弱点打击。
- 工程技能：部署临时装置、修复、过载、短时护盾或诱导敌人。
- 化工技能：投放净化剂、腐蚀弹、冷却剂、催化剂或稳定场。
- 探测技能：扫描资源、标记敌人弱点、揭示污染边界或发现遗迹线索。
- 生存技能：短时抗性、快速撤离、应急治疗或环境适应。

技能消耗可以使用能量、冷却、弹药、药剂或模块耐久。不同消耗类型应和基地后勤形成关系。

## Build 取舍

Build 的目标不是让玩家只追求最高伤害，而是围绕远征目标做准备。

推荐 Build 维度：

- 战斗型：更高输出和控场，适合清理敌巢、Boss 或秘境战斗。
- 探索型：更强扫描、背包、移动和采集效率，适合远距离勘探。
- 防护型：更强环境抗性和容错，适合污染区、腐蚀区或未知区域。
- 工程型：更依赖部署物、工具和特殊弹药，适合处理事故、装置和区域目标。

这些方向可以混搭，不应过早锁死职业。首版可以使用装备、模块和少量技能点形成轻量 Build，而不是完整职业体系。

## 与基地生产的咬合

角色成长和基地生产至少应形成以下关系：

- 武器升级需要金属、晶体、催化材料或遗迹部件。
- 防护升级需要过滤材料、抗性材料、稳定剂或污染处理副产物。
- 特殊弹药需要流体、中间体、能量晶体或生物样本。
- 药剂和临时抗性来自样本分析、过滤和生物处理。
- 探测器和扫描模块需要电子零件、晶体材料和区域样本。
- 区域通行能力需要基地解锁的装备模块或消耗品。

这意味着基地不是单纯建筑升级树，而是角色 Build 的供应链。

## 与地图和区域的咬合

不同区域应要求不同准备，而不是只提高敌人血量。

示例：

- 晶体矿脉区：需要采集工具、扫描模块和中等战斗能力。
- 污染边界区：需要过滤模块、净化剂和污染体应对能力。
- 腐蚀区域：需要抗腐蚀防护、密封剂和耐腐蚀装备材料。
- 生物催化区域：需要孢子过滤、样本隔离和群体敌人处理能力。
- 遗迹或秘境：需要探测模块、稳定剂、强战斗 Build 和充足补给。

区域要求应通过任务、地图图层、装备提示和危险反馈提前告知玩家，避免玩家在不知情的情况下被硬性惩罚。

## 掉落与制造边界

推荐原则：

- 怪物和秘境主要掉落样本、部件、蓝图、核心和特殊材料。
- 完整装备可以少量出现，但不应取代基地制造。
- 高级装备应由掉落材料和基地工艺共同完成。
- 随机词条可以远期评估，首版不应依赖复杂刷词条作为核心乐趣。

这样可以保留 ARPG 奖励感，同时不削弱化工基地的核心地位。

## 首版推荐范围

首版或 MVP 前后建议只验证最小闭环：

1. 两到三种武器或工具风格。
2. 一套防护服或装甲升级线。
3. 少量清晰模块，例如过滤、扫描、采集、战斗增强。
4. 一批基地制造的消耗品，例如治疗、抗性、特殊弹药。
5. 一个区域通行门槛，由防护模块或消耗品解决。
6. 一次秘境或高危区域挑战，验证装备、药剂、战斗和基地后勤的整体关系。

暂不建议首版就做：

- 多职业完整转职。
- 大型天赋树。
- 大量随机词条装备。
- 套装收集。
- 复杂宠物或随从养成。
- 以刷装为主的赛季化终局。

## 当前原型成长状态

截至 2026-05-10，Godot 原型还没有接入完整经验值、职业分支或技能树；当前成长主要通过“任务里程碑 + 基地供给 + 区域适应”三条线体现。

当前已经落地的成长表现是：

- 过滤模块、修复凝胶和抗污染药剂承担早期的环境适应与续战能力。
- 深段之后的每一包推进都要求先把上一跳产物带回基地处理，成长更接近“工业后勤升级”而不是刷怪堆经验。
- 前线回传锚点与基地相位回投台会在深段里程碑后接入，井心室前还会增加第二前线锚点，直接降低晚期纯跑图成本。
- 井系桥东侧的锚场回稳窗会把最新一轮成长收益落成“基地产物改变前线目标结构”，而不是只继续追加新的采集门槛。

当前生命 / 防护成长也已经从固定基础值改成按深段里程碑逐步提高：

- 基础生命 / 防护上限从 `100 / 100` 起步。
- 完成深段样块、裂相尖塔、相位井锁、内层相位井、井底裂口、井心室、井纺室和井纹架断面等关键里程碑时，每次生命上限与防护上限各 `+15`。
- 以当前已接入的成长里程碑计算，晚期角色最高可成长到 `220 / 220`，不再让更东侧战斗长期建立在早期基础耐久上。

这意味着当前原型里的成长收益已经不只是“做出更高阶物品”，而是会同时改变：

- 玩家能否继续进入下一段区域。
- 玩家能否在晚期战斗里保有足够容错。
- 玩家是否需要把基地整备、回投和补给当成有效策略，而不是只靠有限旧图材料硬撑。

当前仍未进入正式范围的内容包括：

- 完整技能树或职业体系。
- 大量随机词条装备。
- 系统化经验条和刷怪成长主轴。

短期设计目标仍然是先把“基地生产出的补给、模块、加工产物和回投能力会真实改变远征结果”这件事验证扎实，再决定是否继续扩成更重的 RPG 成长系统。

## 数据字段预留

未来角色、装备、技能和模块应结构化，至少预留：

- 稳定 ID。
- 显示名。
- 类型。
- 标签。
- 描述文本。
- 装备槽位。
- 基础属性。
- 环境抗性。
- 技能或效果引用。
- 制造配方。
- 升级材料。
- 解锁条件。
- 适用区域或危险类型。
- 公开等级。

这些字段应能被游戏、Wiki 和官方工具共同使用。

官方工具未来可以支持：

- 装备材料反查。
- 区域推荐装备。
- 药剂和特殊弹药查询。
- Build 方案记录。
- 技能、模块和资源链之间的反查。

## 设计边界

- 不把角色成长做成脱离基地的传统刷怪升级。
- 不把基地科技树做成自动替代角色选择。
- 不用复杂词条堆叠掩盖核心循环不清。
- 不让装备门槛变成无提示的硬墙。
- 不让首版 RPG 系统规模超过原型验证需要。

## 当前阶段结论

角色成长、装备、技能和 Build 的核心任务，是把基地生产和外出冒险紧密咬合。

首版应优先验证：

**基地生产补给和装备 -> 玩家形成远征 Build -> 进入危险区域 -> 带回样本和材料 -> 解锁更强装备与工艺 -> 推进更深区域。**

只要这条关系成立，后续才值得扩展更完整的技能树、装备词条、职业差异和终局 Build。
