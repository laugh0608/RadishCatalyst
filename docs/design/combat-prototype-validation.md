# Combat And Interaction Prototype - 原型验证与状态

返回：[Combat And Interaction Prototype](combat-and-interaction-prototype.md)

## 污染与环境压力

首版外勤原型中的污染应至少影响一种交互：

- 持续消耗防护。
- 提高敌人强度或密度。
- 限制建造。
- 需要采样后才能分析。
- 需要净化剂、过滤模块或污染处理设施降低。

污染节点可以是可采样、可破坏或可治理对象，但不应全部相同。

示例：

- 污染沉积物：可采集并带回处理。
- 污染膜：可清理以开放道路或建造面。
- 污染源点：需要基地制造的稳定剂或过滤设备处理。
- 污染巢穴：需要战斗和治理共同解决。

## 首版交互切片

推荐第一条可玩切片：

1. 玩家从基地平台出发。
2. 进入晶体矿脉区。
3. 用扫描器识别矿物、可破坏障碍和污染边界。
4. 采集晶体和浅层矿物。
5. 遭遇基础敌人并获得样本或污染残留。
6. 清理一处小型障碍，打开回路或捷径。
7. 返回基地加工资源。
8. 制造过滤模块、净化剂或基础地板。
9. 平整并铺设一小块地基，扩建加工设备。
10. 使用新模块或药剂进入污染边界深处。
11. 击败精英节点或治理污染源点。
12. 解锁更危险区域的入口。

这条切片能同时验证：

- 移动和战斗。
- 采集和采样。
- 可破坏 / 不可破坏边界。
- 建造前置条件。
- 地形和地基。
- 基地加工反哺外勤。
- 污染作为危险和资源。

## 数据字段预留

地图对象未来应尽量结构化，至少预留：

- 稳定 ID。
- 显示名。
- 对象类型。
- 交互类型。
- 是否可破坏。
- 是否可采集。
- 是否可采样。
- 是否阻挡移动。
- 是否阻挡建造。
- 地形高度等级。
- 地面稳定性。
- 建造需求。
- 破坏需求。
- 掉落表或样本表。
- 污染影响。
- 存档状态。
- Wiki 公开等级。

这些字段应服务游戏、存档、地图 UI、玩家 Wiki 和官方工具。

## 原型验收标准

战斗与交互原型完成时，应能证明：

- 玩家能理解地图上哪些东西能互动，哪些不能互动。
- 玩家能通过工具采集、采样、破坏和清障。
- 玩家能通过基地制造结果改变外勤选择。
- 玩家能理解为什么某处不能建造，并知道如何变成可建造。
- 地形高低差和地基规则能影响基地扩建，但不会成为繁琐负担。
- 污染既是外勤危险，也是基地加工和区域推进的一部分。
- 所有关键交互都能被保存、加载，并为未来联机命令化预留。

## 当前原型状态

截至 2026-05-13，Godot 临时切片已经接入：

- 玩家移动、交互提示、晶体采集、异常采样和污染沉积物采集。
- 前哨核心恢复、晶体簇采集、异常晶体采样、粗糙地块清理、污染沉积斑采集、受扰掠行体击败和封锁遗迹入口确认后的场景内完成态反馈。
- 基础攻击、基础敌人、污染区基础敌人、敌人 HP、击败反馈、敌人掉落和敌人状态记录。
- 敌人即时反击压力：普通敌人造成生命压力，污染敌人额外造成受过滤模块倍率影响的防护压力。
- 生命或防护耗尽后的自动撤回前哨、恢复状态和再尝试建议。
- 污染沉积斑采集会造成防护消耗，基础过滤模块可降低该消耗。
- 最小清理、地基铺设、污染过滤器建造、建造失败原因和污染边界通行门槛。
- 基础反应器和污染过滤器加工：输入消耗、加工中进度、完成入包、最近完成结果和缺料提示。
- 三槽位原型存档 UI、默认槽位快捷键、新档重置、清空槽位、最近 3 份备份、坏档备份恢复和关键运行状态复验。
- HUD 已显示当前目标、目标进度、生命、防护、模块、快捷栏、关键物资、小地图 / 区域标记、交互提示、设备面板雏形、任务完成反馈、撤离反馈、补给反馈和日志。
- HUD 文案和视图数据已拆入独立 presenter，覆盖调试面板、设备面板、任务完成 / 撤离 / 补给反馈、状态摘要、小地图 / 区域标记、任务短提示和日志格式化；`PrototypeHud` 主要保留控件赋值和面板显示胶水。
- 前线回传锚点、基地相位回投台、多已部署落点切换和锚场稳定窗口已进入后段交互原型；`S0` 新档冷启动已验证可从 0 玩到当前主线终点。

仍未完成：

- 精英节点或小型首领挑战仍未实装，当前污染区只有基础受扰敌人承担后段压力。
- 正式战斗手感、受击演出、死亡演出和撤离节奏仍是雏形。
- 建造仍是最小规则，不包含正式建造面板、预览、拖拽放置、管线、电网或复杂地形改造。
- 设备界面仍是基础反应器和污染过滤器的雏形，不包含完整背包转移、自动物流或多设备管理。
- 基础零件缺料提示已能指向前哨核心回收、阶段补给批次、富晶残脉和相位中继锚点后的污染浆液回收线；后段单件制造往返已先收敛为整备包，下一步重点是让现场目标继续产生差异。

## 当前阶段结论

地形、可破坏对象和地基系统值得从首版原型开始考虑，但必须控制粒度。

当前推荐结论是：

**MVP 先做清楚的可破坏 / 不可破坏、可建造 / 不可建造、简单高低差、平整和基础地板 / 地基；暂不做复杂地形编辑、结构承重和多层建筑。**

这样既能让建造和地图交互更有工业基地感，也不会把首版 MVP 的主要风险从核心闭环转移到复杂地形系统。
