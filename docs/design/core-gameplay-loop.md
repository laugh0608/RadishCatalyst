# Core Gameplay Loop

更新时间：2026-05-18

## 文档目的

这份文档用于把项目上位方向转成后续 MVP、原型和系统设计都能引用的核心循环。

它不是功能清单，也不规定具体数值。它只回答：

- 玩家为什么反复出门。
- 玩家为什么反复回基地。
- 化工基地、探索战斗、角色成长和区域解锁如何互相推动。
- 哪些循环关系是首版必须守住的。

## 核心循环一句话

**基地准备远征，远征带回样本和资源，基地把它们转化为装备、工艺和通行能力，再推动玩家进入更危险的新区域。**

这条循环必须比单独的“建工厂”或“刷怪掉材料”更强。

## 主循环

1. 基地准备  
   玩家整理库存、制造补给、检查装备、处理污染、确认设备状态。

2. 选择目标  
   玩家根据当前任务、资源缺口、区域危险和基地需求，决定去哪里探索。

3. 外出探索  
   玩家进入野外、晶体矿脉、污染区、遗迹入口或秘境节点，进行采集、扫描、战斗和调查。

4. 风险应对  
   玩家处理怪物、环境危险、污染、氧气或防护余量、背包容量和撤离路线。

5. 带回成果  
   玩家带回原料、流体样本、生物材料、蓝图、设备核心、遗迹线索或污染残留。

6. 基地加工  
   基地通过反应、分离、储运、供能和污染处理，把成果转化为可用材料、装备、药剂、模块或新工艺。

7. 成长解锁  
   玩家获得更强装备、更稳定生产线、更好的环境适应能力和新的区域通行条件。

8. 进入下一层危险  
   新区域带来更高价值资源、更强生态反击、更复杂污染和更关键的主线线索。

## 三个互锁子循环

### 冒险循环

目标：

- 探路。
- 采样。
- 战斗。
- 回收。
- 撤离。

玩家在冒险循环中要持续判断：

- 继续深入还是返回。
- 采资源还是打敌人。
- 消耗补给还是保留撤离余量。
- 先调查主线还是先补基地短板。

### 基地循环

目标：

- 加工。
- 储运。
- 供能。
- 排障。
- 治理污染。

基地循环的重点不是让玩家一直盯着后台数字，而是让玩家感到基地是下一次远征的后勤核心。

基地应该回答：

- 这次带回来的东西能做什么。
- 下一次远征缺什么。
- 哪条生产链是瓶颈。
- 哪个污染或事故风险正在积累。

### 成长循环

目标：

- 角色更强。
- 装备更适配环境。
- 基地更稳定。
- 区域推进更深。

成长不应只来自打怪经验，也应来自：

- 工艺解锁。
- 样本分析。
- 设备升级。
- 防护和药剂改进。
- 新区域勘探。

## 世界压力循环

为了避免基地成为无风险后台，世界应对玩家行为有反馈：

- 基地污染升高，会改变附近生态和敌袭压力。
- 高价值资源开采，会引发更强的区域反应。
- 设备事故，会制造短期危机或任务后果。
- 秘境调查，会推动主线并打开更危险的路径。

这类压力应该让玩家重视污染处理、安全设施和远征准备，但不能让系统变成惩罚过重的维护游戏。

## 首颗星球的推荐节奏

首颗星球应承担教学和定调作用。

推荐节奏：

1. 建立基础前哨，恢复能源和反应器。
2. 扫描附近晶体矿脉，建立第一条稳定资源链。
3. 发现污染区，建设污染处理设施。
4. 进入污染边界采样，遭遇受扰生态。
5. 用样本解锁过滤模块、抗性补给或特殊加工。
6. 推进到遗迹入口或高价值秘境节点。

这一节奏不等同于 MVP 清单，只是后续拆功能时的方向参照。

## 当前原型验证状态

截至 2026-05-18，Godot 原型已经把这条循环从首切片继续验证到井系桥东侧的锚场回稳窗口、稳窗相位序校准、三条轻量前线行动和第一轮基地行动二选一。当前重点不再是继续横向堆相位井门禁包，也不再停留在二选一验证本身，而是建设基地行动调度与整备系统，让行动反馈改变下一次出发准备。

当前已经稳定跑通的实际节奏已经从“单件门禁制造”转向“基地整备后直接执行现场目标”：

1. 前线回传锚点之前仍保留较直观的验证结构：
   在基地解析上一跳带回的关键样本或遗迹件，得到新的推进锚点。
   向东进入一段更危险的新区域，击败一个新敌人并回收两处新资源点。
   带着新产物返回更东侧终点，勘验后带回下一跳关键成果。
2. 前线回传锚点之后，裂相碎屑、故障残渣、井涌碎屑和相位井后段 6 组装配件已经改为整备包：
   回基地或处理点时一次完成过滤中间件和装配件。
   下一趟外勤直接进入尖塔、井锁、内层相位井、井底裂口、井心室、井纺室、井纹架、井系桥或锚场回稳等现场目标。
   旧的单独组装任务只作为兼容旧存档的 legacy 路径保留，不再作为主线节奏模板。
3. 相位井锚核之后，锚场回稳和稳窗校准继续验证另一种后段循环：
   回基地把锚核解析成归谱片和锚核落尘。
   在同一次锚场整备里稳定锚核落尘，并组装井系校锚桩。
   把基地产物带回前线部署，清理压力钉并压住井系守脉体，再完成锚场回稳。
   以局部稳定窗口和相位井余响片作为现场收益，再回基地无基础零件税解析成稳窗读数，并按西侧、中央、东侧顺序校准稳窗相位序。
4. `S16` 后的轻量前线行动已经把完成态收益转成三轮短回访，并在 `S20` 之后验证行动前取舍：
   玩家先连续完成稳窗回波、补给回执和巡线信标三条短行动，再从基地选择稳场补给、相位测绘或压力清障；补给只读一处低风险投放点，测绘读两处前线测绘点，清障只清一处高风险压力扰点。
   选择反馈随后进入统一行动台：补给转成低风险补给计划，测绘转成信息侦测计划，清障转成防护整备计划，玩家确认出发整备槽后才由相位回投台执行。

当前最深已经验证到：

```text
外圈回波 -> 深段门禁 -> 深段样块 -> 前线回传锚点
-> 内层故障轨迹 -> 相位井定位器 -> 相位井芯样本
-> 相位井心核 -> 相位井纺核 -> 相位井织核
-> 相位井结核 -> 相位井锚核 -> 相位井余响片
-> 相位井稳窗读数 -> 稳窗相位序校准 -> 三轮轻量前线行动
-> 基地行动选择 -> 相位测绘反馈 -> 出发整备确认 -> 基础风险收益计划
```

这说明当前原型里的“基地加工改变下一次外勤结果”已经不再只体现在早期过滤模块和抗污染药剂上，也体现在：

- 更东侧区域逐段解锁。
- 前线回传锚点与基地相位回投台缩短晚期纯跑图。
- 深段生命 / 防护上限随里程碑成长，避免晚期长期停在固定基础值。
- 基础零件供给从有限晶体簇扩展到阶段补给、富晶残脉和污染浆液回收，不再由早期矿点决定后段主线能否推进。
- 基地产出的前线稳场工具会直接改变下一次外勤的目标结构，稳窗读数和稳窗校准会把完成态转成可回访容错收益，并进一步引出轻量前线行动、行动前取舍、出发整备确认和基础风险收益计划。

短期设计约束也已明确：

- 继续优先用单张横向切片验证更东侧内容，不急着切新场景。
- 继续复用污染过滤器和基础反应器，不为每一包内容临时引入第三台设备。
- 回投、成长、局部稳定窗口、轻量前线行动、基地行动选择、出发整备槽和风险收益计划都属于循环收益的一部分，不能重新退回“只加材料，不改远征节奏”的旧口径。
- `S0` 新档冷启动和 `S5` 到 `S16` 首轮复测已支撑前一阶段收口；`S17` 到 `S20` 已支撑基地-前线-基地短循环、行动前取舍、整备确认和基础风险收益通过处理，下一步转向行动队列或清障分支。

## 多星球扩展循环

后续如果引入不同星球，每颗星球都应复用同一套主循环，但改变问题类型：

- 腐蚀星球：重点是耐腐蚀材料、酸雾治理和泄漏风险。
- 赤铁荒漠星球：重点是高温、粉尘、散热和矿物开采。
- 冰封挥发物星球：重点是低温流体、保温、冻结管线和储罐安全。
- 生物催化星球：重点是样本隔离、孢子污染、生物反应和生态反击。
- 遗迹区域：重点是异常能量、古代工艺核心、秘境战斗和主线真相。

## 设计边界

- 不把基地做成纯后台生产表。
- 不把探索做成单纯打怪掉材料。
- 不让化工复杂度变成专业考试。
- 不让战斗系统脱离基地后勤。
- 不让联机成为首版循环成立的前提。

## 对后续文档的要求

后续 `mvp-feature-list.md`、`resources-and-process-chain.md`、`maps-and-regions.md` 和角色成长文档，都应能指回这条循环。

如果某个功能无法说明它如何强化“基地准备 -> 外出探索 -> 带回成果 -> 基地加工 -> 成长解锁 -> 深入危险区域”，就应谨慎进入首版范围。
