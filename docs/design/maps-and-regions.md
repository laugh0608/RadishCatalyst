# Maps And Regions

更新时间：2026-05-13

## 文档目的

这份文档用于定义 RadishCatalyst 的地图、区域、星球和秘境在项目方向中的角色。

它不是地图清单，也不是关卡制作计划。它只先固定：

- 地图层级如何组织。
- 首颗星球应承担什么作用。
- 区域如何服务探索、化工、污染和主线。
- 时间与天气如何服务环境压力和区域差异。
- 多星球扩展如何保持差异但不打散项目识别度。

## 地图层级

建议使用四层结构理解地图内容：

1. 星球或大型区域  
   决定主地貌、生态、污染类型、资源体系和长期探索主题。

2. 区域  
   承载一组资源、危险、敌人、任务和基地适应需求。

3. 地点  
   具体的资源点、污染点、遗迹入口、敌人据点、前哨节点或事故现场。

4. 交互节点  
   可采集物、样本、可破坏障碍、设备残骸、任务线索、环境机关、怪物巢穴或秘境入口。

这套层级应方便后续存档、地图 UI、Wiki 和官方工具引用。

## 首颗星球定位

首颗星球应是项目标准体验的基准。

它需要同时展示：

- 明亮但危险的异星环境。
- 可读的化工基地。
- 晶体矿脉、基础资源和少量污染。
- 外出探索与基地建设的互相依赖。
- 第一批主线异常和秘境入口。

推荐视觉基调：

- 冷白日光。
- 浅色岩地。
- 蓝绿色晶体生态。
- 少量黄色污染区。
- 清晰的管线、储罐、反应器、电力塔和污染处理设施。

## 首颗星球区域类型

### 基地平台

玩家的核心据点。

功能：

- 建设反应器、储罐、泵站、管线、电力和污染处理设施。
- 整理库存和远征补给。
- 查看生产、污染、能源和当前目标。

视觉要求：

- 清晰、稳定、可扩建。
- 工程感强，但不压抑。
- 与周围异星地貌形成明确边界。

### 晶体矿脉区

首批高价值资源来源。

功能：

- 提供基础晶体矿物或催化材料。
- 引导玩家建立第一条资源链。
- 让玩家理解采集、扫描、回收和加工的关系。

视觉要求：

- 蓝绿色晶体具有辨识度。
- 资源点在地图上容易识别。
- 可以带少量原生生物或环境危险。

### 污染边界区

连接化工基地和世界反馈的区域。

功能：

- 展示污染指数、污染扩散和污染治理。
- 提供污染残留、异常样本或受扰生态。
- 迫使玩家准备防护、过滤或治理设备。

视觉要求：

- 污染边界清楚。
- 危险可读，不能只靠大面积雾效。
- 治理前后应有可见差异。

### 未探索生态区

冒险和资源扩展区域。

功能：

- 放置新资源、新怪物、地貌障碍和调查点。
- 让玩家体验基地外的未知感。
- 承接区域任务和主线线索。

视觉要求：

- 与基地平台形成自然过渡。
- 生态要有异星感，但不应让可交互物难以辨认。

### 遗迹入口或秘境节点

首颗星球的主线钩子。

功能：

- 提供高风险目标。
- 解锁新工艺、新设备或关键剧情。
- 验证玩家的装备、药剂和基地后勤准备。

视觉要求：

- 与自然生态和人造基地都不同。
- 有明确的异常感。
- 可以作为远期主线与多星球扩展的入口。

## 当前原型区域带状图

截至 2026-05-13，当前可玩地图仍是一张单向向东扩展的横向切片，但区域职责已经被验证到更晚段内容。

当前原型的区域推进顺序是：

```text
基地平台
-> 晶体矿脉区
-> 处理点 / 污染边界
-> 封锁遗迹入口
-> 遗迹外圈
-> 深段门禁与阵列段
-> 相位井锁前段
-> 内层相位井
-> 井底裂口
-> 井心室断面
-> 井纺室断面
-> 井纹架断面
-> 井系桥断面与锚场回稳窗
-> 稳窗相位序校准 -> 稳窗回波探点
```

这条带状图在当前原型中承担了几条明确规则：

- 裂相脊到锚场已开始拆掉“一怪、两物、一个终点”的复制模板：前段补共振读数、脉冲读数、泄压和清余烬壳，井心室 / 井纺室 / 井系桥补现场读数解锁采集，井纹架补侧路清障，锚场补部署后的压力钉清理与短守场。
- 稳窗读数不会再打开新区域，而是把锚场完成态校准成可回访回充点；稳窗相位序校准和回波探点继续复用同一区域，验证短回访而不是扩图。
- 新区域的进入条件不靠单纯数值墙，而靠上一跳基地解析、过滤或组装后的产物来开启。
- 小地图当前已经覆盖基地、晶体、污染、外圈、深段、井口、井底、心室、井纺、井纹和井系十一段 marker，用来支撑晚期长链路的目标可读性。
- 区域任务索引、地图交互实例和 HUD 目标区域当前都按场景真实落点同步维护，避免“任务说往东，但地图没标或标错区”。

当前地图里的回投与落点口径也已经进入设计真相源：

- 深段读数矩阵完成后，深段固定点会开放第一前线回传锚点。
- 进入井心室段后，地图最东侧会再开放第二前线锚点，作为当前最新深段落点。
- 基地相位回投台仍只返回最近校准的当前锚点，不扩成全图自由传送网。

短期仍维持这条单切片向东扩展结构，不急着拆成多场景跳转；`S0` 新档冷启动和 `S5` 到 `S16` 首轮复测已支撑前一阶段收口，下一步应把同一区域转成基地-前线行动短闭环，而不是继续向东扩图。

## 多星球与区域差异原则

不同星球应复用统一的基地、UI 和数据语言，但改变当地问题。

示例：

- 腐蚀星球：酸雾、盐壳、腐蚀湖、耐腐蚀材料和泄漏治理。
- 赤铁荒漠星球：红褐矿层、粉尘风暴、高温散热和重矿开采。
- 冰封挥发物星球：低温储罐、冻结管线、挥发物采集和保温设备。
- 生物催化星球：菌毯、孢子、生物反应池、样本隔离和生态反击。
- 遗迹区域：几何结构、异常能量、古代工艺核心和高风险秘境。

每个星球至少应回答：

- 这里有什么独特资源。
- 这里有什么独特污染或环境危险。
- 这里有什么典型时间或天气压力。
- 玩家需要哪些基地或装备准备。
- 这里的怪物和生态为什么存在。
- 这里如何推进主线或长期目标。

## 时间与天气系统

时间和天气系统应成为环境压力的一部分，而不是纯视觉装饰。

长期可以支持：

- 白天 / 夜晚。
- 晴朗、阴天、雨、雪、风暴。
- 星球特有天气，例如酸雾、金属粉尘、低温挥发、孢子雨、异常能量潮。

时间与天气可以影响：

- 能见度。
- 敌人活动。
- 污染扩散或沉降。
- 角色防护消耗。
- 设备稳定性。
- 发电、冷却、过滤、排水或保温需求。
- 可采集资源或样本出现窗口。

首版边界应保持克制：

- 至少为时间和天气在地图、存档和 UI 中预留字段。
- 如果实现，优先做昼夜切换或一种区域天气。
- 天气必须有清晰提示，并允许玩家通过装备、药剂、模块或基地设施应对。
- 不做复杂季节、全球气候模拟、随机灾害链或长期不可控惩罚。

## 地图 UI 图层

地图应预留以下图层：

- 已探索 / 未探索。
- 基地范围。
- 资源点。
- 污染边界。
- 当前时间和天气。
- 危险等级。
- 敌对生态活动。
- 任务目标。
- 秘境和遗迹入口。
- 前哨、传送点或补给点。

地图图层应服务玩家决策，而不是只做装饰。

## 地形与建设关系

地图不只是移动背景，也应影响基地建设和外勤交互。

首版建议保留以下基础判断：

- 地形或原有建筑可分为可破坏与不可破坏。
- 地块可分为可建造与不可建造。
- 地形高低差会影响设备、管线、地板和大型设施的放置。
- 部分地块需要先清理、平整或铺设地板 / 地基，才能承载基地设备。
- 不可破坏大型地貌、主线遗迹和地图边界用于保护关卡节奏和存档稳定。

这部分详细规则以 `combat-and-interaction-prototype.md` 为准。当前阶段只确定方向：地形应该参与基地扩建和探索决策，但首版不做复杂地形雕刻、结构承重或多层建筑。

## 数据字段预留

区域和地点未来应尽量结构化，至少预留：

- 稳定 ID。
- 显示名。
- 星球或区域归属。
- 地貌类型。
- 地形高度等级。
- 可建造性。
- 可破坏性。
- 资源列表。
- 污染类型。
- 时间状态。
- 天气类型。
- 危险等级。
- 推荐防护。
- 关联设备或工艺。
- 怪物或生态类型。
- 任务和主线关联。
- Wiki 公开等级。

## 当前阶段结论

首颗星球应先作为项目视觉、玩法和叙事的基准，不急着铺开大量星球。

多星球是长期扩展方向，但每个新星球都必须带来新的资源、污染、生态、基地需求或主线问题，而不只是换色地图。
