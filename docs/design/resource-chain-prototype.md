# Resources And Process Chain - 当前原型链路

返回：[Resources And Process Chain](resources-and-process-chain.md)

## 当前阶段结论

资源链的重点不是数量，而是关系。

首版之前应先确定一条能闭合的链路：

**外出采集或战斗获得资源 -> 基地加工和治理污染 -> 产出探索/战斗能力 -> 解锁更危险区域 -> 获得更高价值资源。**

这条链路稳定后，再进入具体 MVP 配方和数值拆分。

当前加工已经从即时命令推进到最小 `in_progress` 运行状态：启动时消耗输入，设备记录当前配方和加工进度，完成后才发放产物 / 副产并记录最近完成结果。基础反应器和污染过滤器已有设备面板雏形，能显示当前配方、输入、产出、副产、耗时、进度、缺料、最近完成和下一步建议；基础反应器会按当前任务自动优先选择目标配方，降低后期配方变多后只能靠手动轮换查找的摩擦。

当前这些链路的价值已经不只是“做出一个新物品”，而是会继续落到：

- 更东侧区域解锁。
- 新敌人和新采集点的进入许可。
- 回基地再出发的补给和过滤准备。
- 更晚期的前线回投落点前移与深段生存曲线承接。
- 副产物转主输入，以及“把基地产物带回前线部署”这种新的任务结构。
- 完成态读数和稳窗相位序校准把前线稳定窗口变成可回访回充点，并进一步承接第一版轻量前线行动回报，而不是只增加一件库存纪念物。

`S0` 新档冷启动已于 2026-05-13 验证可从 0 玩到当前主线终点。基础零件后段供给不再依赖有限晶体簇，短期由关键前线阶段的前哨核心回收 / 阶段补给批次承接；`晶体矿物 -> 基础零件` 保留为早期教学和应急加工，不作为后段主线预算的唯一来源。晶体矿脉区北侧 `富晶残脉` 作为可选兜底，相位中继锚点后的 `回收基础零件` 可把污染浆液低效率转回基础零件。后段解析类配方已移除固定基础零件税，装配类配方继续消耗基础零件，但主线会把装配件并入整备包，避免回基地只做一个零件。`scripts/check-client-data.ps1` 已新增主线资源预算检查，并随 `check-client.ps1` 聚合执行，覆盖新档初始库存、主线采集、采样、中间加工、建筑消耗、任务件部署和 3 个基础零件误用缓冲。

当前原型也验证了一条重要约束：即使内容一路扩到锚场回稳窗口和第一版轻量前线行动，短期仍可以继续只复用污染过滤器和基础反应器，而不急着为了“像工厂游戏”临时引入第三台设备。当前仍不包含管线、电网、储罐、自动物流、完整背包转移、多设备管理或复杂设备调度；下一步不应主动扩展复杂工艺链，而应让基地确认、前线短目标和回基地反馈形成可延展模板。
