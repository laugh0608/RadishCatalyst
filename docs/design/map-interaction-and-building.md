# Combat And Interaction Prototype - 地图交互与建造

返回：[Combat And Interaction Prototype](combat-and-interaction-prototype.md)

## 地图对象交互分类

地图上的对象至少应分成以下几类：

### 可采集

玩家使用工具或交互键获得资源。

示例：

- 晶体矿脉。
- 浅层矿物。
- 植被或菌毯样本。
- 污染沉积物。

### 可采样

玩家获得样本、线索或解锁信息，不一定直接获得大量资源。

示例：

- 异常反应点。
- 污染边界。
- 受扰生态残留。
- 遗迹入口装置。

### 可破坏

玩家可以通过武器、工具、爆破或特定装备移除。

示例：

- 小型晶体簇。
- 脆弱岩块。
- 受污染的生物膜。
- 已废弃的轻型设施残片。
- 可清理的障碍物。

可破坏对象应带来至少一种结果：

- 打开路径。
- 获得资源。
- 降低危险。
- 触发敌人或污染反应。
- 暴露样本或入口。

### 不可破坏

玩家不能通过常规手段移除。

示例：

- 大型地貌边界。
- 深坑、峭壁或主地形骨架。
- 主线遗迹核心结构。
- 基地平台关键锚点。
- 用于限制原型范围的区域边界。

不可破坏对象不是失败设计。它们用于保护地图可读性、关卡节奏、存档稳定和后续任务结构。

### 可建造

玩家可以放置地板、地基、设备、储存、管线或电力设施。

可建造性应由地形、空间、污染、坡度、高低差和已有对象共同决定。

### 不可建造

玩家不能直接建造。

原因可以包括：

- 地面不平。
- 高低差过大。
- 被资源、建筑残骸或敌对巢穴占据。
- 污染过高。
- 属于保护区域、遗迹区域或任务区域。
- 缺少地基、支撑或前置清理。

## 地形与高低差

用户补充的“地形高低差、平整土地和地板 / 地基单元”非常适合进入建造交互边界。

推荐首版采用离散层级，而不是连续地形模拟：

- 地图格或地块可以有高度等级，例如低地、平地、高地。
- 同一设备占地范围内需要高度一致。
- 管线、地板和设备不应跨越过大的高度差。
- 玩家可以通过平整、填平、架高地板或放置地基来获得可建造区域。
- 高低差可影响移动路径、敌人路径和基地扩建方向。

这样既能让地形参与玩法，又不会把首版拖进复杂体素、真实土方或物理施工系统。

## 平整与地基

首版建议保留四种建设前置动作：

1. 清理  
   移除小型障碍、资源残留、污染膜或废弃设施。

2. 平整  
   把指定地块整理为可建造状态，消耗基础材料或工具时间。

3. 铺设地板 / 地基  
   创建稳定建造面，允许放置轻型设备、管线、储存和工作台。

4. 强化地基  
   作为可选扩展，用于支持重型设备、储罐、反应器或高风险工艺。

MVP 必做不应超过：

- 简单清理。
- 简单平整。
- 一种基础地板或地基单元。
- 建造时显示能否放置以及失败原因。

可选扩展：

- 不同地基等级。
- 斜坡、桥梁或支架。
- 污染地块需要先净化再建造。
- 大型设备需要强化地基。
- 地形改造会影响污染扩散或敌人路径。

暂不做：

- 真实土方体积计算。
- 大规模地形雕刻。
- 多层建筑。
- 复杂结构承重。
- 完整水体、流沙、坍塌或滑坡模拟。

## 建造放置规则

首版建造放置至少检查：

- 是否在允许建造区域。
- 占地范围是否为空。
- 地面是否已清理。
- 高低差是否满足要求。
- 是否有地板 / 地基或天然稳定地面。
- 污染是否低于设备允许阈值。
- 是否拥有材料。
- 是否与已有管线、设备或电力连接规则冲突。

失败提示应明确，例如：

- 地面不平。
- 缺少地基。
- 污染过高。
- 被障碍物占据。
- 距离基地核心过远。
- 材料不足。

这些提示是首版 UI 的一部分，因为它们能让玩家理解地形、资源和基地扩建的关系。
