# Narrative And Quest Framework - 任务框架规则

返回：[Narrative And Quest Framework](narrative-and-quest-framework.md)

## 支线任务职责

支线任务不应在首版承担大量剧情分支。它们主要服务四类目标：

### 教学补强

示例：

- 尝试平整一块地。
- 制作第二份药剂。
- 使用扫描器标记资源。
- 修复一个储存箱。

### 资源补给

示例：

- 回收前哨残骸。
- 清理晶体簇。
- 采集额外污染沉积物。

### 世界反馈

示例：

- 污染升高后附近敌人变多。
- 处理污染膜后开出捷径。
- 某处天气或夜间能见度变化影响探索。

### 主线铺垫

示例：

- 发现旧前哨日志。
- 扫描遗迹材料。
- 分析受扰生态样本。

## 任务类型

首版任务类型建议限制在少量可实现类型：

- 前往地点。
- 采集资源。
- 采样对象。
- 击败敌人。
- 建造或启用设备。
- 部署前线装置并完成现场稳定。
- 制造物品。
- 使用物品。
- 解锁区域。
- 返回基地。

暂不做：

- 多分支对话。
- 护送。
- 大规模防守波次。
- 复杂潜行。
- 随机任务系统。
- 声望和阵营任务。

## 任务与系统解锁

任务应推动系统逐步出现：

1. 基地库存和设备。
2. 外出采集。
3. 基础战斗。
4. 样本分析。
5. 配方解锁。
6. 地基和建造前置。
7. 装备 / 药剂 / 模块。
8. 污染区通行。
9. 精英挑战。
10. 后续入口。

不要一次性把所有 UI、设备、配方和地图图层都抛给玩家。

## 叙事信息载体

首版推荐使用：

- 任务目标。
- 短日志。
- 前哨系统提示。
- 样本分析结果。
- 地图标记。
- 设备异常提示。
- 少量无线电或终端文本。

暂不需要：

- 复杂过场动画。
- 大段配音。
- 多角色长对话。
- 完整世界编年史。

## 命名和语气

任务命名应短、明确、偏现场行动。

示例：

- 恢复前哨供能。
- 勘探晶体矿脉。
- 分析异常样本。
- 建立过滤能力。
- 进入污染边界。
- 稳定污染源点。

语气原则：

- 工业开拓。
- 危险但理性。
- 少玄学，多现场证据。
- 不把化工术语堆成专业考试。

## 首版不做

首版叙事暂不做：

- 完整主线全章。
- 多结局。
- 大型剧情分支。
- 多阵营声望。
- 大量 NPC。
- 恋爱、家园社交或经营剧情。
- 完整配音。
- 过长文本演出。

## 当前阶段结论

首版故事应围绕一个问题展开：

**这座前哨为什么必须恢复，污染边界里到底发生了什么。**

任务框架只需要支撑第一条闭环，而不是提前写完整长篇剧情。

只要任务能自然引导玩家完成：

**恢复基地 -> 外出采集和战斗 -> 分析样本 -> 制造通行能力 -> 进入污染区 -> 发现更大异常**

首版叙事就足以支撑 Godot 原型。
