# Character Progression And Equipment - 原型状态与边界

返回：[Character Progression And Equipment](character-progression-and-equipment.md)

## 当前原型成长状态

截至 2026-05-13，Godot 原型还没有接入完整经验值、职业分支或技能树；当前成长主要通过“任务里程碑 + 基地供给 + 区域适应”三条线体现。

当前已经落地的成长表现是：

- 过滤模块、修复凝胶和抗污染药剂承担早期的环境适应与续战能力。
- 深段之后的每一包推进都要求先把上一跳产物带回基地处理，成长更接近“工业后勤升级”而不是刷怪堆经验。
- 前线回传锚点与基地相位回投台会在深段里程碑后接入，井心室前还会增加第二前线锚点，直接降低晚期纯跑图成本。
- 井系桥东侧的锚场回稳窗会把最新一轮成长收益落成“基地产物改变前线目标结构”，稳窗读数还会把完成态转成可回访的生命 / 防护回充点。

当前生命 / 防护成长也已经从固定基础值改成按深段里程碑逐步提高：

- 基础生命 / 防护上限从 `100 / 100` 起步。
- 完成深段样块、裂相尖塔、相位井锁、内层相位井、井底裂口、井心室、井纺室和井纹架断面等关键里程碑时，每次生命上限与防护上限各 `+15`。
- 以当前已接入的成长里程碑计算，晚期角色最高可成长到 `220 / 220`，不再让更东侧战斗长期建立在早期基础耐久上。

这意味着当前原型里的成长收益已经不只是“做出更高阶物品”，而是会同时改变：

- 玩家能否继续进入下一段区域。
- 玩家能否在晚期战斗里保有足够容错。
- 玩家是否需要把基地整备、回投和补给当成有效策略，而不是只靠有限旧图材料硬撑。

当前仍未进入正式范围的内容包括：

- 完整技能树或职业体系。
- 大量随机词条装备。
- 系统化经验条和刷怪成长主轴。

短期设计目标仍然是先把“基地生产出的补给、模块、加工产物和回投能力会真实改变远征结果”这件事验证扎实，再决定是否继续扩成更重的 RPG 成长系统。

## 数据字段预留

未来角色、装备、技能和模块应结构化，至少预留：

- 稳定 ID。
- 显示名。
- 类型。
- 标签。
- 描述文本。
- 装备槽位。
- 基础属性。
- 环境抗性。
- 技能或效果引用。
- 制造配方。
- 升级材料。
- 解锁条件。
- 适用区域或危险类型。
- 公开等级。

这些字段应能被游戏、Wiki 和官方工具共同使用。

官方工具未来可以支持：

- 装备材料反查。
- 区域推荐装备。
- 药剂和特殊弹药查询。
- Build 方案记录。
- 技能、模块和资源链之间的反查。

## 设计边界

- 不把角色成长做成脱离基地的传统刷怪升级。
- 不把基地科技树做成自动替代角色选择。
- 不用复杂词条堆叠掩盖核心循环不清。
- 不让装备门槛变成无提示的硬墙。
- 不让首版 RPG 系统规模超过原型验证需要。

## 当前阶段结论

角色成长、装备、技能和 Build 的核心任务，是把基地生产和外出冒险紧密咬合。

首版应优先验证：

**基地生产补给和装备 -> 玩家形成远征 Build -> 进入危险区域 -> 带回样本和材料 -> 解锁更强装备与工艺 -> 推进更深区域。**

只要这条关系成立，后续才值得扩展更完整的技能树、装备词条、职业差异和终局 Build。
