# Resources And Process Chain

更新时间：2026-05-13

## 文档目的

这份文档用于定义 RadishCatalyst 的资源与化工链路原则。

它不是配方表，也不是首版资源清单。它先回答：

- 资源为什么值得玩家出门获取。
- 化工链为什么是游戏特色而不是背景设定。
- 固体、流体、副产物、污染和设备如何服务冒险。
- 后续数据、Wiki 和官方工具应如何组织这些关系。

## 核心原则

资源链必须服务主循环：

**外出获得危险资源，基地通过化工链把它们转成探索能力、战斗能力、基地能力和主线推进能力。**

因此资源不能只用于堆建筑数量，也不能只用于卖钱。

## 资源类型

### 固体资源

作用：

- 建材。
- 设备零件。
- 装甲材料。
- 催化载体。
- 遗迹残片。

设计重点：

- 容易被玩家理解。
- 适合做基础建设和装备升级。
- 可通过采矿、回收和敌人掉落获得。

### 流体资源

作用：

- 溶剂。
- 冷却剂。
- 燃料。
- 反应介质。
- 污染液。
- 生物培养液。

设计重点：

- 流体应是项目差异化重点，不只是固体资源的另一种图标。
- 流体需要储罐、泵、管线、压力或容量管理。
- 流体可以带来泄漏、污染、冻结、挥发或腐蚀风险。

### 生物与异常样本

作用：

- 解锁新工艺。
- 制作药剂、过滤模块、特殊弹药或生物催化剂。
- 推进主线调查。

设计重点：

- 样本来自探索和战斗。
- 样本需要分析、稳定或隔离。
- 样本不应只变成普通材料，应和剧情、生态和工艺解锁有关。

### 能源与中间体

作用：

- 驱动基地。
- 支撑高级工艺。
- 制造装备和远征补给。

设计重点：

- 中间体让生产链有层次。
- 能源瓶颈推动玩家升级基地。
- 中间体和能源都应能被 Wiki 和工具反查。

## 化工链表达方式

推荐链路结构：

1. 来源  
   区域采集、怪物掉落、污染回收、遗迹回收或基地副产物。

2. 预处理  
   粉碎、过滤、洗涤、冷却、加热、稳定或分级。

3. 反应或分离  
   通过反应器、分离器、储罐、泵站、换热器、过滤器等设备完成关键转化。

4. 产物  
   得到材料、药剂、装备模块、燃料、催化剂或任务样本。

5. 副产物与风险  
   产生废液、废气、污染、热量、压力、堵塞或不稳定中间体。

6. 回流利用  
   部分副产物可以进入污染处理、能源回收或高级工艺。

这能让化工系统有特色，但不要求玩家学习真实公式。

## 首颗星球资源链方向

首颗星球建议围绕以下资源关系建立基准，而不是一开始铺大量资源：

- 蓝绿色晶体矿脉：提供基础催化矿物、能量晶体或高级材料前体。
- 浅层矿物和金属残片：提供建材、管线、设备外壳和基础零件。
- 污染液或污染沉积物：既是风险来源，也是可处理回收的特殊材料。
- 水或基础溶剂：支撑洗涤、冷却、过滤和基础反应。
- 原生生态样本：用于药剂、过滤模块、生物催化或主线分析。

第一批链路应体现：

- 固体加工。
- 流体储运。
- 反应或分离。
- 污染处理。
- 一种能直接反哺外出探索的产物。

例如：

- 晶体矿物经过粉碎和反应，产出基础催化材料。
- 污染液经过过滤和中和，降低基地污染，并回收少量高活性副产物。
- 生物样本经过稳定处理，解锁过滤模块或抗性药剂。

这些例子不是最终配方，只是方向约束。

## 产物必须反哺冒险

基地应能产出：

- 武器和装备材料。
- 防护服模块。
- 抗腐蚀、抗毒、抗寒或抗孢子药剂。
- 特殊弹药。
- 探测器和扫描组件。
- 便携补给和维修包。
- 区域通行所需的过滤器、稳定剂或能量核心。

如果某条生产链只让基地数字变大，却不能改变玩家的探索、战斗、区域进入或主线推进，它就不应成为首版重点。

## 污染与副产物

污染不是单纯惩罚。

它应同时承担：

- 风险反馈。
- 资源回收。
- 区域变化。
- 怪物和生态变化。
- 主线调查线索。

副产物设计原则：

- 低级副产物可以先通过污染处理稳定。
- 中级副产物可以回收为燃料、溶剂或催化材料。
- 高级副产物可能关联秘境、异常生态或主线真相。

## 设备角色

首批设备方向：

- 反应器：核心转化。
- 储罐：流体缓冲和安全边界。
- 泵站：流体输送。
- 管线：连接设备和表达流程。
- 过滤器：污染治理、样本处理和前处理。
- 分离器：产物、副产物和废物流区分。
- 电力塔或发电装置：支撑基地运行。
- 污染处理设施：连接基地安全和区域生态。

设备 UI 应重点表达：

- 输入。
- 输出。
- 副产物。
- 当前状态。
- 处理速率。
- 容量。
- 异常原因。

## 数据与工具要求

资源、配方和设备未来应结构化，方便游戏、Wiki 和官方工具共享。

建议预留字段：

- 稳定 ID。
- 显示名。
- 类型。
- 标签。
- 来源。
- 输入材料。
- 输出材料。
- 副产物。
- 所需设备。
- 能源消耗。
- 污染影响。
- 解锁条件。
- 公开等级。

官方工具至少应能反查：

- 某资源从哪里来。
- 某资源能做什么。
- 某设备能处理什么。
- 某配方会产生哪些副产物。
- 某区域值得带什么装备或药剂。

## 当前阶段结论

资源链的重点不是数量，而是关系。

首版之前应先确定一条能闭合的链路：

**外出采集或战斗获得资源 -> 基地加工和治理污染 -> 产出探索/战斗能力 -> 解锁更危险区域 -> 获得更高价值资源。**

这条链路稳定后，再进入具体 MVP 配方和数值拆分。

当前加工已经从即时命令推进到最小 `in_progress` 运行状态：启动时消耗输入，设备记录当前配方和加工进度，完成后才发放产物 / 副产并记录最近完成结果。基础反应器和污染过滤器已有设备面板雏形，能显示当前配方、输入、产出、副产、耗时、进度、缺料、最近完成和下一步建议；基础反应器会按当前任务自动优先选择目标配方，降低后期配方变多后只能靠手动轮换查找的摩擦。

当前这些链路的价值已经不只是“做出一个新物品”，而是会继续落到：

- 更东侧区域解锁。
- 新敌人和新采集点的进入许可。
- 回基地再出发的补给和过滤准备。
- 更晚期的前线回投落点前移与深段生存曲线承接。
- 副产物转主输入，以及“把基地产物带回前线部署”这种新的任务结构。
- 完成态读数把前线稳定窗口变成可回访回充点，而不是只增加一件库存纪念物。

`S0` 新档冷启动已于 2026-05-13 验证可从 0 玩到当前主线终点。基础零件后段供给不再依赖有限晶体簇，短期由关键前线阶段的前哨核心回收 / 阶段补给批次承接；`晶体矿物 -> 基础零件` 保留为早期教学和应急加工，不作为后段主线预算的唯一来源。`scripts/check-client-data.ps1` 已新增主线资源预算检查，并随 `check-client.ps1` 聚合执行，覆盖新档初始库存、主线采集、采样、中间加工、建筑消耗、任务件部署和 3 个基础零件误用缓冲。

当前原型也验证了一条重要约束：即使内容一路扩到锚场回稳窗口，短期仍可以继续只复用污染过滤器和基础反应器，而不急着为了“像工厂游戏”临时引入第三台设备。当前仍不包含管线、电网、储罐、自动物流、完整背包转移、多设备管理或复杂设备调度；下一步不应主动扩展复杂工艺链，而应先补齐基础零件缺料提示、可选补给指引和前线回传锚点后的任务节奏重构。
