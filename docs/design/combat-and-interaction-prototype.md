# Combat And Interaction Prototype

更新时间：2026-05-04

## 文档目的

这份文档用于把首版 MVP 中的外勤、战斗、地图交互和建造前置条件拆成 Godot 原型可执行边界。

它不是完整战斗设计，也不是完整建造系统方案。它只回答：

- 首版原型里玩家能做哪些动作。
- 地图上的资源、建筑、地形和敌人如何被交互。
- 哪些对象可破坏，哪些对象不可破坏。
- 建造为什么需要考虑地形高低差、平整和地基。
- 原型阶段哪些内容必须做，哪些只做预留。

## 原型目标

首版战斗与交互原型应验证：

**玩家从基地出发，在地图上移动、探索、采集、采样、战斗、清障、撤离，并把带回的资源交给基地加工。**

它的目标不是做出完整动作游戏手感，也不是做复杂地形工程，而是确认玩家在外勤过程中始终有清楚判断：

- 这里能不能走。
- 这里能不能采。
- 这里能不能破坏。
- 这里能不能建。
- 这里危险来自敌人、污染、地形，还是资源准备不足。
- 我现在应该继续深入、清障、采样，还是撤回基地。

## 玩家基础动作

原型必须支持：

- 移动。
- 面向或瞄准。
- 基础攻击。
- 使用工具。
- 采集资源。
- 采样调查。
- 拾取掉落物。
- 使用快捷物品。
- 与基地设备或地图节点交互。
- 回到基地或撤离点。

可选支持：

- 闪避或短距离冲刺。
- 简单击退。
- 简单蓄力工具动作。
- 临时扫描。

首版不要求复杂连招、空中动作、格挡反击或高精度动作取消。

## 战斗最小边界

必须实现：

- 一种通用武器或工具攻击。
- 一种近程或远程基础敌人。
- 一种更危险的精英、巢穴或小型首领挑战。
- 生命、防护和能量 / 体力中的至少两个压力来源。
- 受击反馈、死亡 / 失败反馈和撤离后保留成果的规则。
- 敌人掉落样本、部件或污染残留，而不是主要掉完整装备。

推荐首版敌人类型：

- 原生小型生物：用于验证基础移动、攻击和采集干扰。
- 受扰污染体：用于验证污染区、样本掉落和防护压力。
- 精英节点守卫：用于验证装备、药剂、特殊弹药或模块准备。

暂不做：

- 大量敌人类型。
- 复杂 Boss 阶段。
- 职业技能循环。
- 大规模弹幕。
- PvP。
- 多人同步战斗。

## 工具与武器关系

首版可以把武器和工具先做成少量可复用交互能力。

推荐：

- 工程步枪：通用攻击，验证远程战斗。
- 切割 / 冲击工具：可攻击，也可破坏部分障碍或采集矿物。
- 扫描器：标记资源、样本、危险边界或可破坏对象。
- 投放器：使用净化剂、特殊弹药或稳定剂处理污染节点。

工具不应只为战斗服务。它们应同时连接：

- 资源采集。
- 环境清障。
- 样本调查。
- 基地制造产物。
- 区域通行门槛。

## 地图对象交互分类

地图上的对象至少应分成以下几类：

### 可采集

玩家使用工具或交互键获得资源。

示例：

- 晶体矿脉。
- 浅层矿物。
- 植被或菌毯样本。
- 污染沉积物。

### 可采样

玩家获得样本、线索或解锁信息，不一定直接获得大量资源。

示例：

- 异常反应点。
- 污染边界。
- 受扰生态残留。
- 遗迹入口装置。

### 可破坏

玩家可以通过武器、工具、爆破或特定装备移除。

示例：

- 小型晶体簇。
- 脆弱岩块。
- 受污染的生物膜。
- 已废弃的轻型设施残片。
- 可清理的障碍物。

可破坏对象应带来至少一种结果：

- 打开路径。
- 获得资源。
- 降低危险。
- 触发敌人或污染反应。
- 暴露样本或入口。

### 不可破坏

玩家不能通过常规手段移除。

示例：

- 大型地貌边界。
- 深坑、峭壁或主地形骨架。
- 主线遗迹核心结构。
- 基地平台关键锚点。
- 用于限制原型范围的区域边界。

不可破坏对象不是失败设计。它们用于保护地图可读性、关卡节奏、存档稳定和后续任务结构。

### 可建造

玩家可以放置地板、地基、设备、储存、管线或电力设施。

可建造性应由地形、空间、污染、坡度、高低差和已有对象共同决定。

### 不可建造

玩家不能直接建造。

原因可以包括：

- 地面不平。
- 高低差过大。
- 被资源、建筑残骸或敌对巢穴占据。
- 污染过高。
- 属于保护区域、遗迹区域或任务区域。
- 缺少地基、支撑或前置清理。

## 地形与高低差

用户补充的“地形高低差、平整土地和地板 / 地基单元”非常适合进入建造交互边界。

推荐首版采用离散层级，而不是连续地形模拟：

- 地图格或地块可以有高度等级，例如低地、平地、高地。
- 同一设备占地范围内需要高度一致。
- 管线、地板和设备不应跨越过大的高度差。
- 玩家可以通过平整、填平、架高地板或放置地基来获得可建造区域。
- 高低差可影响移动路径、敌人路径和基地扩建方向。

这样既能让地形参与玩法，又不会把首版拖进复杂体素、真实土方或物理施工系统。

## 平整与地基

首版建议保留四种建设前置动作：

1. 清理  
   移除小型障碍、资源残留、污染膜或废弃设施。

2. 平整  
   把指定地块整理为可建造状态，消耗基础材料或工具时间。

3. 铺设地板 / 地基  
   创建稳定建造面，允许放置轻型设备、管线、储存和工作台。

4. 强化地基  
   作为可选扩展，用于支持重型设备、储罐、反应器或高风险工艺。

MVP 必做不应超过：

- 简单清理。
- 简单平整。
- 一种基础地板或地基单元。
- 建造时显示能否放置以及失败原因。

可选扩展：

- 不同地基等级。
- 斜坡、桥梁或支架。
- 污染地块需要先净化再建造。
- 大型设备需要强化地基。
- 地形改造会影响污染扩散或敌人路径。

暂不做：

- 真实土方体积计算。
- 大规模地形雕刻。
- 多层建筑。
- 复杂结构承重。
- 完整水体、流沙、坍塌或滑坡模拟。

## 建造放置规则

首版建造放置至少检查：

- 是否在允许建造区域。
- 占地范围是否为空。
- 地面是否已清理。
- 高低差是否满足要求。
- 是否有地板 / 地基或天然稳定地面。
- 污染是否低于设备允许阈值。
- 是否拥有材料。
- 是否与已有管线、设备或电力连接规则冲突。

失败提示应明确，例如：

- 地面不平。
- 缺少地基。
- 污染过高。
- 被障碍物占据。
- 距离基地核心过远。
- 材料不足。

这些提示是首版 UI 的一部分，因为它们能让玩家理解地形、资源和基地扩建的关系。

## 污染与环境压力

首版外勤原型中的污染应至少影响一种交互：

- 持续消耗防护。
- 提高敌人强度或密度。
- 限制建造。
- 需要采样后才能分析。
- 需要净化剂、过滤模块或污染处理设施降低。

污染节点可以是可采样、可破坏或可治理对象，但不应全部相同。

示例：

- 污染沉积物：可采集并带回处理。
- 污染膜：可清理以开放道路或建造面。
- 污染源点：需要基地制造的稳定剂或过滤设备处理。
- 污染巢穴：需要战斗和治理共同解决。

## 首版交互切片

推荐第一条可玩切片：

1. 玩家从基地平台出发。
2. 进入晶体矿脉区。
3. 用扫描器识别矿物、可破坏障碍和污染边界。
4. 采集晶体和浅层矿物。
5. 遭遇基础敌人并获得样本或污染残留。
6. 清理一处小型障碍，打开回路或捷径。
7. 返回基地加工资源。
8. 制造过滤模块、净化剂或基础地板。
9. 平整并铺设一小块地基，扩建加工设备。
10. 使用新模块或药剂进入污染边界深处。
11. 击败精英节点或治理污染源点。
12. 解锁更危险区域的入口。

这条切片能同时验证：

- 移动和战斗。
- 采集和采样。
- 可破坏 / 不可破坏边界。
- 建造前置条件。
- 地形和地基。
- 基地加工反哺外勤。
- 污染作为危险和资源。

## 数据字段预留

地图对象未来应尽量结构化，至少预留：

- 稳定 ID。
- 显示名。
- 对象类型。
- 交互类型。
- 是否可破坏。
- 是否可采集。
- 是否可采样。
- 是否阻挡移动。
- 是否阻挡建造。
- 地形高度等级。
- 地面稳定性。
- 建造需求。
- 破坏需求。
- 掉落表或样本表。
- 污染影响。
- 存档状态。
- Wiki 公开等级。

这些字段应服务游戏、存档、地图 UI、玩家 Wiki 和官方工具。

## 原型验收标准

战斗与交互原型完成时，应能证明：

- 玩家能理解地图上哪些东西能互动，哪些不能互动。
- 玩家能通过工具采集、采样、破坏和清障。
- 玩家能通过基地制造结果改变外勤选择。
- 玩家能理解为什么某处不能建造，并知道如何变成可建造。
- 地形高低差和地基规则能影响基地扩建，但不会成为繁琐负担。
- 污染既是外勤危险，也是基地加工和区域推进的一部分。
- 所有关键交互都能被保存、加载，并为未来联机命令化预留。

## 当前原型状态

截至 2026-05-05，Godot 临时切片已经接入：

- 玩家移动、交互提示、晶体采集、异常采样和污染沉积物采集。
- 前哨核心恢复、晶体簇采集、异常晶体采样、粗糙地块清理、污染沉积斑采集、受扰掠行体击败和封锁遗迹入口确认后的场景内完成态反馈。
- 基础攻击、基础敌人、污染区基础敌人、敌人 HP、击败反馈、敌人掉落和敌人状态记录。
- 敌人即时反击压力：普通敌人造成生命压力，污染敌人额外造成受过滤模块倍率影响的防护压力。
- 生命或防护耗尽后的自动撤回前哨、恢复状态和再尝试建议。
- 污染沉积斑采集会造成防护消耗，基础过滤模块可降低该消耗。
- 最小清理、地基铺设、污染过滤器建造、建造失败原因和污染边界通行门槛。
- 基础反应器和污染过滤器加工：输入消耗、加工中进度、完成入包、最近完成结果和缺料提示。
- 三槽位原型存档 UI、默认槽位快捷键、新档重置、清空槽位、最近 3 份备份、坏档备份恢复和关键运行状态复验。
- HUD 已显示当前目标、目标进度、生命、防护、模块、快捷栏、关键物资、小地图 / 区域标记、交互提示、设备面板雏形、任务完成反馈、撤离反馈、补给反馈和日志。
- HUD 文案和视图数据已拆入独立 presenter，覆盖调试面板、设备面板、任务完成 / 撤离 / 补给反馈、状态摘要、小地图 / 区域标记、任务短提示和日志格式化；`PrototypeHud` 主要保留控件赋值和面板显示胶水。

仍未完成：

- 精英节点或小型首领挑战仍未实装，当前污染区只有基础受扰敌人承担后段压力。
- 正式战斗手感、受击演出、死亡演出和撤离节奏仍是雏形。
- 建造仍是最小规则，不包含正式建造面板、预览、拖拽放置、管线、电网或复杂地形改造。
- 设备界面仍是基础反应器和污染过滤器的雏形，不包含完整背包转移、自动物流或多设备管理。
- 人工冷启动试玩仍需继续验证玩家是否能无解释完成第一切片，而不是只依赖自动化流程复验。

## 当前阶段结论

地形、可破坏对象和地基系统值得从首版原型开始考虑，但必须控制粒度。

当前推荐结论是：

**MVP 先做清楚的可破坏 / 不可破坏、可建造 / 不可建造、简单高低差、平整和基础地板 / 地基；暂不做复杂地形编辑、结构承重和多层建筑。**

这样既能让建造和地图交互更有工业基地感，也不会把首版 MVP 的主要风险从核心闭环转移到复杂地形系统。
