# Vertical Slice - 原型状态与风险

返回：[Vertical Slice](vertical-slice.md)

## 当前原型状态

截至 2026-04-29，临时原型已经具备：

- `Boot` / `GameRoot` 启动链。
- 首批 JSON 静态数据和 `DataRegistry`。
- `WorldState`、`CharacterState`、`InventoryState`、`QuestState` 默认状态。
- 一张 `VerticalSliceMap` 临时地图，包含基地平台、晶体区和污染区色块。
- 玩家移动、前哨核心交互、晶体采集、异常采样、污染沉积物采集和粗糙地块清理。
- 前哨核心恢复后会保留可见并切换为已恢复颜色，提供第一步任务完成后的场景内反馈。
- 晶体簇采集、异常晶体采样和粗糙地块清理后会保留可见完成态，切换颜色和标签并关闭重复交互；运行刷新、读档和新档重置会重新应用对应状态。
- `GatherSystem` 规则层雏形，交互节点不再直接发奖励，并已校验静态数据中的工具标签。
- 基础攻击输入、近距离敌人命中、敌人 HP 变化、击败反馈和敌人状态记录。
- 敌人击败后会按原型掉落数据发放物品，并记录掉落已发放状态，避免重复领取。
- 敌人受击后会进行即时反击，普通敌人造成生命压力，污染敌人额外造成会受过滤模块倍率影响的防护压力。
- 生命或防护耗尽后，玩家会自动撤回前哨并恢复到可继续尝试的状态；正式撤离结果面板会区分生命耗尽、防护耗尽或二者都耗尽，并在 HUD 面板和日志中提示修复凝胶、抗污染药剂、过滤模块或回基地补给。
- `CharacterState.quick_slots` 已记录快捷栏绑定，玩家可用 `1` 使用修复凝胶恢复生命，用 `2` 使用抗污染药剂恢复防护；HUD 左侧已接入快捷栏绑定面板，可在修复凝胶、抗污染药剂 I 和空槽之间轮换两个槽位；使用补给、满状态、缺少物品或空槽时会显示短反馈，提示恢复量、当前生命 / 防护、剩余数量或回基地加工建议。
- `SaveService` 支持通过 `K` 保存、`L` 读取默认槽位，当前保存世界状态、角色状态、背包、快捷栏、任务、地图对象、敌人和建筑状态；读取时会校验存档版本、世界块和角色块，并把内容一致性、地图对象 / 敌人 / 建筑来源、配方与区域解锁来源、任务链、任务目标进度来源和首批跨块关系复验交给 `SaveContentValidator`；文件缺失、JSON 解析失败、版本不兼容、关键块缺失或运行时内容不一致时会给出中文失败提示，并保留当前运行状态；保存时会先轮转最近 3 份 `.bak.1` / `.bak.2` / `.bak.3` 备份。
- 原型 HUD 左侧已接入 3 个存档槽位入口，支持查看空槽 / 可读取 / 备份可恢复状态，并通过按钮保存、读取或清空 `slot_01`、`slot_02` 和 `slot_03`；面板顶部新增“新档”入口，可不依赖初始存档直接重置当前运行进度；`K` / `L` 仍保留为默认 `slot_01` 的调试快捷键。
- `check-client-save.ps1` 已补充切片结尾状态复验，覆盖污染边界完成后污染残核目标仍处于 active、污染残核压制后外圈勘探目标仍处于 active，以及确认外圈中继后 `slice_01_complete` 解锁、角色位置和区域状态可随存档恢复；存档运行时复验代码已拆为 `save_service_check.gd` 总入口、地图对象 / 敌人来源 suite、建筑运行时 / 设备缓冲 suite、任务目标 suite 和任务解锁效果 suite，后续新增跨块关系复验应继续按职责拆分。
- `check-client-flow.ps1` 已作为第一切片关键流程复验入口，覆盖恢复前哨、晶体目标、样本回收、过滤模块、处理点扩建、污染边界、污染残核、封锁遗迹入口、遗迹外圈第二闭环和 `slice_01_complete` 的任务链推进。
- `check-client-data.ps1` 已覆盖任务链数据维护护栏，包括配方解锁来源、任务激活来源、区域任务索引、主线任务图一致性和任务目标来源；`check-client-scenes.ps1` 已覆盖第一切片任务目标与 `VerticalSliceMap.tscn` 中交互点、建造点、加工设备、敌人实例和地图对象掉落数量的对应关系。
- `ProcessingSystem` 规则层雏形，基础反应器可在处理晶体矿物、组装反应器校准件、分析异常样本、制备过滤介质、调制修复凝胶、组装基础过滤模块和制造基础地基材料之间轮换，并会按当前任务自动优先选择目标配方；启动加工时消耗输入并写入设备 `in_progress` 状态，完成后才发放产物 / 副产并推进任务目标，日志会说明产物 / 副产已放入背包和下一步建议。
- 靠近基础反应器或污染过滤器时，HUD 交互面板会显示设备名、当前配方、输入、产出、副产、耗时、加工进度、最近完成配方、入库去向、下一步建议、可加工 / 缺料 / 未解锁状态和切换配方提示。
- 基础过滤模块最小启用命令，玩家可用 `F` 将背包中的 `equipment.filter_module_t1` 装入防护服模块槽，降低污染沉积斑交互造成的防护消耗。
- 污染沉积斑采集会提示本次防护消耗，以及过滤模块是否已降低消耗；低防护进入污染边界时会给出补给警告。
- 污染沉积斑采集后会保留“已回收”状态，受扰掠行体击败后会显示“污染已压制”，封锁遗迹入口确认后会保留“信号已确认”状态，使污染边界后段也能通过场景对象确认推进结果。
- HUD 已增加当前区域污染状态提示，会显示区域污染强度、过滤模块是否生效、防护消耗倍率和低防护建议；污染边界被挡回时会补充处理点扩建、启用过滤模块或补给防护等具体原因。
- `BuildSystem` 最小建造规则，支持清理粗糙地块后铺设 2 块基础地基，并消耗材料建造污染过滤器。
- 建造点 / 地块状态提示已复用 `BuildSystem` 的前置与缺料判断，靠近粗糙地块、基础地基建造点或污染过滤器建造点时，HUD 会显示清理状态、材料需求、基础地基数量、缺料 / 前置原因和可执行操作。
- 交互、攻击、配方切换、加工、建造和区域门槛的失败结果已补充轻量结构化提示，日志会显示失败类型、原因和下一步建议。
- 污染边界区污染过滤器建造完成后才启用，可执行 `recipe.cleanse_residue`，将污染沉积物和基础溶剂处理为抗污染药剂并产出污染浆液副产物；该设备复用建造点生成的 `building.pollution_filter` 结构状态，不额外生成临时设备实例。
- 污染边界区基础敌人 `enemy.polluted_skitter`，拥有更高 HP、污染区颜色区分和击败反馈。
- 污染边界区精英节点 `enemy.elite_residue_node` 已接入主线，完成污染边界任务后出现；击败后才解锁封锁遗迹入口，使过滤模块、修复凝胶和抗污染药剂在末段挑战中有直接价值。

- 完整任务系统、教程提示和更正式的任务完成界面表现；当前只完成任务完成短时反馈雏形。
- 正式多槽位管理、设备运行时来源和更完整的跨块存档内容一致性复验；下一轮不主动扩展这部分，除非新增真实可保存状态或暴露实际坏档问题。

## 正式设备面板雏形

第一轮冷启动试玩已经证明任务链路可推进，设备面板雏形只解决“设备当前在做什么、为什么不能做、下一步该按什么”的理解断点，不先做大范围 UI 重构。

最小实现范围：

- 只覆盖基础反应器和污染过滤器。
- 复用现有设备交互入口，在玩家靠近设备时用 `Q` 打开 / 关闭设备详情面板；右侧日志和 HUD 状态继续保留为原型调试信息。
- 面板显示设备名称、当前运行状态、当前配方、可切换配方列表、输入 / 产出 / 副产、加工耗时、加工进度、最近完成结果、背包缺料项和下一步操作。
- 配方切换和启动加工继续走现有 `ProcessingSystem`、配方解锁、设备状态和缺料判断，不新增平行规则。
- 默认不新增存档字段；除非后续确认为必要，不保存面板打开状态或光标选择状态。

暂不纳入：

- 全局 HUD 重构。
- 拖拽式背包、仓库转移或批量投料。
- 电网、管线、自动物流和多设备调度。
- 正式小地图、任务完成界面、撤离 / 死亡表现和战斗节奏调整。

## 非目标

第一切片不追求：

- 画面最终品质。
- 完整数值平衡。
- 完整剧情表现。
- 全部装备系统。
- 完整自动化工厂。
- 完整天气系统。
- 联机。
- 大规模内容量。

## 风险控制

如果实现中遇到延期，优先保留：

1. 基地 -> 外出 -> 回基地 -> 再深入的闭环。
2. 存档和稳定 ID。
3. 任务目标和引导。
4. 采集、加工、制造、通行门槛。
5. 一种敌人和一种污染压力。

可以先砍：

- 第二种武器。
- 复杂天气。
- 多种地基。
- 多条支线。
- 额外敌人。
- 装饰性 UI。
- 继续加深存档坏档边界。
- 更多解释性 HUD 文本。

## 当前阶段结论

Godot 工程初始化后，不应先做“通用大框架”，而应围绕这条切片搭最小可运行结构。

只要第一切片能证明：

**外出获得资源和样本 -> 基地加工 -> 产出远征能力 -> 进入更危险区域**

项目就具备继续扩展的基础。

当前下一轮不再围绕小修和护栏收敛打转，而应直接推进下一包 demo 内容。任务链数据、目标来源、区域归属和场景实例已有检查可继续守住状态链路；后续只有在新内容开发暴露真实问题时，才按需补强这些护栏。

## 遗迹外圈第一版

当前已把“遗迹外圈”第一版接到现有主线上，用来验证第一切片之后的第二条可玩闭环，而不是只把现有地图继续拉长。

本轮已包含：

- 在封锁遗迹入口之后补一段新的可走区域或更大的连续地图段。
- 至少加入 1 个新的外勤阻挡 / 危险，以及 1 个新的资源、调查点或推进目标。
- 优先复用基础反应器和污染过滤器，产出 1 个能明显改变下一次深入结果的开路物、环境对策、补给或通行能力。
- 形成“外出受阻 -> 回基地加工 -> 再次深入 -> 达成新目标”的第二闭环。
- 在第二闭环成立后，把外圈深段收益继续落成“守卫战 -> 回波匣 -> 回基地分析 -> 更深坐标”的延伸价值，而不是只停在结尾提示。

这包当前仍暂不做：

- 为此提前扩写新的通用大系统。
- 只放大地图面积但不增加新的风险、收益和基地回报。
- 大范围 UI 重构、正式教程框架、电网、管线、自动物流或第三台基地设备；除非内容验证已经明确被它们阻塞。

当前完成判定仍以这三点为准：

- 玩家能明确感受到基地准备改变了遗迹外圈的推进方式，而不是只重复第一切片已有动作。
- 新区域的阻挡、收益和回基地加工价值在无口头解释下仍可理解。
- 新内容若暴露任务链、HUD 或验证缺口，再按需补强，不把预支工程治理当成这包内容的前置。

## 深段往返节奏约束

当前第二轮深段闭环已经证明“回基地加工”本身仍有价值，但也暴露出另一个风险：如果只是继续把深段区域往右拉长，玩家会越来越频繁地为纯移动时间买单。

下一包内容建议按以下约束处理：

- 基地控制的前线回传设施第一版已接入：深段固定部署点上的前线回传锚点已与基地平台的相位回投台连通。
- 当前仍保持“前线 -> 基地快速回传、基地 -> 最近校准锚点回投”的范围，不提供全图自由传送网。
- 当前允许多个前线锚点实例，但基地回投同一时刻只返回 1 个当前锚点；回传能力继续绑定深段读数矩阵和基地加工结果，让“基地服务冒险，冒险反哺基地”保持成立。
- `S12` GUI 人工复测已于 2026-05-11 通过；这说明“继续复用基础反应器 + 污染过滤器”已经足够证明后段闭环成立，下一轮主要风险转为玩法同构，而不是链路能不能跑通。
- 默认禁止继续复制“一个门禁 + 一个怪物 + 两处收集 + 过滤器 + 反应器 + 下一门禁”模板。
- 若继续定义相位井锚核之后的新包，至少改变 1 个维度：外勤目标结构、风险 / 战斗关系、资源获取模式、基地加工决策，或前线锚点 / 回投节奏。
- 相位井锚核后的第一包现已按这条约束落地成“前线短守场 + 局部稳定窗口”结构；当前重点观察它是否真正缩短无决策空跑、是否仍保留基地后勤价值，以及是否需要额外提示或护栏。

## 相位井锚核之后的已实现包

这包不再继续证明“新门禁 + 新怪物 + 两处收集”还能再跑一遍，而是改为验证另一种后段推进形态：基地先产出前线稳场工具，下一次外勤的主要目标从搜刮材料改成一次短守场。

当前实现链路：

1. 回基地基础反应器解析 `相位井锚核`，得到 `相位井归谱片`，并产出副产 `锚核落尘`。
2. 回处理点污染过滤器，把 `锚核落尘` 稳定成 `稳场滤囊`。
3. 回基地基础反应器，把 `相位井归谱片`、`稳场滤囊` 和基础零件组装成 `井系校锚桩`。
4. 带着 `井系校锚桩` 返回井系桥东侧固定点，部署到 `锚场回稳窗`。
5. 部署后先清两处 `锚场压力钉`，再触发 `井系守脉体` 短守场；失败后校锚桩保留，玩家可直接重试。
6. 回稳成功后，东侧留下一个可持续的局部稳定窗口，现场回充生命 / 防护，并产出第一份 `相位井余响片`；余响片回基地解析成 `相位井稳窗读数` 后，该窗口会变成可回访的前线回充点。

这包已经故意改变的维度：

- 外勤目标从“沿路回收两处新材料”改成“把基地产物带到现场完成守场目标”。
- 前线收益从“再拿一把门钥匙”改成“制造一个能回充生命 / 防护并改变容错节奏的局部稳定窗口”。
- 敌人价值从“守门素材怪”改成“打断稳场过程的压力源”。

当前实现边界：

- 仍只复用基础反应器和污染过滤器。
- 不新增第三台设备、电网、管线、自动物流或第三永久前线锚点。
- 不做完整波次系统；当前只做一个主压力敌人和回稳交互，不额外引入完整波次刷怪系统。
- 守场失败后不强制玩家回基地重做 `井系校锚桩`；已部署桩保留在现场，允许直接重试。
- 这包完成判定不是地图再向右多一块，而是玩家明确感受到“基地加工先改变任务结构，再改变前线容错”。

当前复测重点：

- `S13` / `S14` / `S15` GUI 复测已通过；`S15` 是完成态回归入口，不是新的玩家任务。
- 当前不再继续沿相位井后段加包；先回到 `S0` 冷启动和里程碑复核，判断首小时阶段是否通过、需回改，还是可进入下一个主要阶段。
