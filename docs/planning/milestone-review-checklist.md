# Milestone Review Checklist

更新时间：2026-04-26

## 文档目的

这份文档用于定义 RadishCatalyst 在每个阶段或里程碑结束时必须进行的复核。

它不是测试用例清单，也不是发布流程。它只回答：

- 每个阶段成果完成后要检查什么。
- 如何避免前期实现拖垮后期扩展和兼容。
- 什么时候必须暂停继续加功能，先修正架构或范围。

## 总原则

每个里程碑都必须回答三个问题：

1. 当前成果是否真的强化了首版闭环。
2. 当前实现是否堵死了未来存档、联机、工具、平台或内容扩展。
3. 当前范围是否仍然符合个人或小团队可执行节奏。

如果答案不清楚，应先复核和修正，不继续堆新内容。

## 里程碑类型

当前建议按以下里程碑推进：

1. 文档地基完成。
2. Godot 工程初始化完成。
3. 第一条静态数据链路跑通。
4. 玩家移动、采集和基础战斗跑通。
5. 基地加工和库存跑通。
6. 建造、地基和地图交互跑通。
7. 污染、防护和区域门槛跑通。
8. 存档保存 / 读取跑通。
9. 第一可玩切片串通。
10. 首小时引导可玩。

每个里程碑完成后，都应做一次复核。

## 必查项目

### 闭环检查

确认：

- 是否服务“基地准备 -> 外出探索 -> 回基地加工 -> 解锁更危险区域”。
- 是否能说明该功能对玩家体验的价值。
- 是否避免了只增加复杂度但不推动闭环的功能。

### 平台检查

确认：

- Windows 桌面版体验没有被 Web 或 Android 约束反向牺牲。
- Web 游戏客户端仍只是可选试玩 / 技术验证。
- Android 仍是远期评估，不作为首版实现压力。
- WebApp 规划只指 Wiki、配方查询器、工艺链规划器、地图 / 资源工具等辅助产品。

### GDScript 与 .NET 边界检查

确认：

- Godot 客户端核心仍以 GDScript 为跨端和原型基线。
- 没有把必须支持 Web 试玩的核心客户端逻辑押到 C#。
- .NET / C# 主要用于外部工具、数据校验、存档迁移、服务端或未来非 Web 客户端可选模块。
- 如果未来引入 C#，必须先更新架构文档和平台兼容评估。

### 存档检查

确认：

- 新增状态是否进入 `save-data-model.md` 中对应边界。
- 是否使用稳定定义 ID 或世界内实例 ID。
- 没有用场景节点路径、显示名或 UI 状态作为存档真相源。
- 新状态能保存、读取、校验，并为迁移预留。

### 静态数据检查

确认：

- 新物品、配方、设备、敌人、区域、任务、天气和污染是否能配置化。
- 数据 ID 是否稳定。
- 显示名和描述是否通过本地化键引用。
- 数据是否能被未来 Wiki 和官方工具复用。

### 工程结构检查

确认：

- 状态在 `state/`。
- 规则在 `systems/`。
- 表现在 `scenes/`、`actors/`、`ui/`。
- 存档在 `save/`。
- 静态数据在 `data/`。
- UI 不直接修改核心状态。
- 没有出现单文件大脚本承载过多职责。

### 任务和教程检查

确认：

- 任务目标清楚。
- 新系统出现前有足够引导。
- 玩家失败后知道如何补救。
- 没有一次性展示过多系统。
- 首小时仍能让玩家完成一次可信闭环。

### 性能和规模检查

确认：

- 当前内容规模没有超过第一可玩切片需要。
- 地图、敌人、设备、天气或污染模拟没有过早膨胀。
- Web 可选试玩不作为性能预算主约束。
- 如果某系统出现明显性能风险，应先简化规则或延后。

### 文档同步检查

确认：

- 若修改了平台、存档、工程结构、验证基线或阶段边界，已同步 `docs/`。
- 若修改了协作规则或默认验证流程，已同步 `AGENTS.md` 和 `CLAUDE.md`。
- 若完成重要推进，已追加到当周周志。

## 暂停信号

出现以下情况时，应暂停新增功能：

- 一个功能无法说明如何服务首版闭环。
- 为了 Web 或 Android 牺牲 Windows 主体验。
- 为了未来 .NET 迁移而阻塞当前可玩原型。
- 存档状态已经开始依赖场景节点路径或 UI 临时变量。
- 静态数据开始硬编码在多个脚本中。
- 任务系统和教程 UI 互相绑定，无法复用。
- 单个脚本接近 1000 行且继续增加职责。
- 一个里程碑没有通过基础验证却继续进入下一个阶段。

## 当前阶段结论

RadishCatalyst 的里程碑复核目标不是增加流程负担，而是避免前期实现把后期兼容和扩展拖垮。

每个里程碑都应先确认：

**闭环仍然成立，状态仍可保存，数据仍可复用，平台边界仍清楚，范围仍可执行。**
