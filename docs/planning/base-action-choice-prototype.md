# Base Action Choice Prototype

更新时间：2026-05-20

## 目标

「基地行动选择原型」验证一件事：玩家在基地确认下一趟短行动前，能从 2 到 3 个方案中做出可感知取舍，并在同一趟前线目标和返回反馈里看到结果。

当前稳场补给 / 相位测绘 / 压力清障三选一已完成首轮自动验证。后续不是继续压缩范围，而是把它升级为「基地行动调度与整备系统」：行动台、队列、清障分支或更完整风险等级都可以进入设计，只要它们服务出发前决策，并且不是为了堆同构短行动。

## 首轮方案表

| 基地行动前选项 | 玩家选择含义 | 前线差异 | 返回收益 | 失败兜底 |
| --- | --- | --- | --- | --- |
| 稳场补给 | 把本趟行动定位为低风险补给回收。 | 前线只读取 / 标记一处补给投放点，不额外刷战斗目标；路线靠近井系桥前线第三锚点。 | 回基地解析为少量 `基础零件`、`修复凝胶` 或下一趟短行动补给，强调基地把前线回执转成整备资源。 | 读点未完成时不消耗选择；返回基地仍保留该方案，可重新回投读取，不造成资源断档。 |
| 相位测绘 | 把本趟行动定位为侦测与路线确认。 | 前线需要读取两处相位测绘点，位置比补给方案更分散，但不强制战斗；HUD 可提示测绘进度。 | 回基地解析为下一趟行动提示、局部目标显形或一次前线回投默认点确认，强调信息收益。 | 只读到一处时给出“粗测记录”，允许回基地解析成较弱提示；另一处保持可读，避免卡任务。 |
| 压力清障 | 把本趟行动定位为高风险清障。 | 前线需要清掉一处压力钉 / 小型扰动体后读取清障回执；风险来自一次短战斗或危险点。 | 回基地解析为额外防护 / 抗性补给、稳定碎片或更高价值回收，强调风险换收益。 | 清障失败不回滚已确认行动；已清目标保留，敌压重试从前线锚点开始，不要求回基地重做。 |

## 首版实现口径

- `S19` 之后已接入一轮选择：基地行动台展示稳场补给 / 相位测绘两个互斥选项，选择后立即进入对应前线短目标。
- 当前行动调度状态已进入 `WorldState`：稳场补给反馈会准备低风险补给计划，相位测绘反馈会准备信息侦测计划；两类反馈会写入当前计划槽，玩家需先在前线行动台确认出发整备槽，再由下一次相位回投消耗。
- 行动队列最小版已接入：行动台会展示当前计划槽和下一计划候选，方案终端可替换下一候选，但不会改动当前已准备的出发槽；本趟相位回投执行后，下一候选会晋升为当前可确认计划，并滚动生成新的下一候选；晋升后的计划可继续确认和回投执行。
- 稳窗回访、补给短行动和巡线短行动的确认目标已统一到前线行动台；旧补给 / 巡线分台仅保留定义与存档兼容，不再作为玩家可见的必经确认点。
- 三个方案共享“基地选择 -> 前线短目标 -> 回基地基础反应器解析反馈”的骨架，只让目标数量、风险和收益文案产生差异。
- 当前首轮已实现 `稳场补给`、`相位测绘` 和 `压力清障`；清障先只清一处前线压力扰点，不引入新战斗系统。
- 返回收益只使用现有资源、补给、HUD 地图 / 任务提示能力；不得为首版新增长期成长货币。

## 后续承接

- 下一步优先补强更明确的风险收益预告；后续扩展必须继续复用已有整备槽、行动候选和风险收益计划。
- 稳场补给反馈应影响下一次补给包、资源缓冲或前线补给标记。
- 相位测绘反馈应影响下一次路线提示、地图标记、回投建议或目标预告。
- 队列、取消 / 改签、队员、装备 loadout、成功率和更完整风险等级不是禁区；只有当它们不能服务当前基地整备目标时才暂缓。
