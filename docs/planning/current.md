## 当前阶段

RadishCatalyst 当前处于「首小时引导可玩」阶段。

第一切片冷启动审计已于 2026-05-05 通过，上一阶段「Godot 第一可玩切片原型骨架实现」退出条件已满足。当前重点不是按分钟数堆内容，而是验证玩家是否能理解并完成：

```text
外出获得资源 -> 回基地加工 -> 远征产物改变下一次外勤结果
```

2026-05-05 空存档试玩显示完整流程扩展后仍不足 10 分钟。该时长仅作为内容密度、理解断点和工程摩擦的观察信号，不再作为阶段硬目标。

## 最近进度

- `client/` 已建立 Godot 4.x 原型工程，第一切片主流程、任务链、静态数据、地图实例、存档运行时、HUD 关键面板和验证脚本已可支撑反复冷启动复测。
- 第一切片已覆盖前哨恢复、晶体采集、基础战斗、基地加工、异常样本分析、基础过滤模块、处理点建设、污染边界挑战、污染残核压制和切片结尾提示。
- 遗迹外圈第一版内容包已接入现有主线：确认封锁遗迹入口后可进入新地图段，回收继电残片、回基地组装稳相信标，再返回稳定抖动雾幕并确认外圈中继。
- 外圈中继后的延伸链已接入：玩家会继续在外圈深段清理相位守卫、回收外圈回波匣，并回基地用基础反应器解析更深遗迹坐标，让“更深入口价值”落到实际战利品和基地分析结果上。
- 基础反应器已支持任务相关配方自动优先选择，设备面板会标记当前目标推荐配方，降低靠 `R` 反复切换查找配方的摩擦。
- HUD 已开始从原型巨型脚本拆分职责：调试面板、设备面板、反馈面板、常驻状态、小地图、短提示和日志格式化已有独立 presenter 承担文本和展示规则；常驻 HUD 现已收口为分散式布局，左上地图、左侧目标、右上状态和左下提示 / 日志已能兼顾信息可见性与主视野；静态数据和场景检查已覆盖主线任务图、任务目标来源和地图实例对应关系。
- 第一切片舞台已从静态摆拍切到 `WorldCamera` 跟随玩家：深段和最右侧区域会随推进进入视野，场景检查也已按相机驱动口径验证热点与 HUD 遮挡关系。
- 最近几轮已收敛任务图、目标来源、区域归属、HUD 关键提示和场景实例等高频分叉点；这些护栏现阶段已足够支撑更快的内容扩展，后续改为随开发进度按需补强，不再作为默认主线。
- 2026-05-08 空存档复测已再次确认：从外圈中继到相位守卫、回波匣和回基地解析深段回波的延伸链无明显阻断，可正式把主线从“验证第二闭环是否成立”转向“把更深遗迹入口做成可落地的新一包内容”。
- 更深遗迹入口第一版已落地：`deep_ruin_coordinates` 会在遗迹外圈最东侧写入深段门禁，打开新的深段入口区域；玩家随后会在深段回收相位纤丝，回基地复用污染过滤器精炼谐振滤芯，再复用基础反应器组装深段覆写栓，返回现场覆写深段锁扣并带回第一份深段样块，让“外出获得资源 -> 回基地加工 -> 远征产物改变下一次外勤结果”继续成立。
- 深段样块后的第二轮闭环已接入：玩家会回基地解析深段样块，整理出深段路由印片；再返回深段点亮阵列台、清理深段追袭体并回收相位导管；最后回基地把相位导管和污染浆液整理成深段读数矩阵，让深段收益继续反哺下一轮推进。
- 前线回传能力第一版已接入：玩家整理深段读数矩阵后，会返回深段固定点部署前线回传锚点；此后可在深段按锚点快速回基地，并在基地通过相位回投台返回最后部署锚点。
- 开发复测基线第一版已接入：调试面板现在可直接载入 `S0` 到 `S8` 九个代码生成的阶段快照，用来替代大多数“每次从空档重打”的人工复测。
- `S4` / `S5` 回传节奏回修已接入：`S4` 现在会保留第二轮深段线继续推进所需的基础零件和污染浆液，`S5` 在基地起跑时 HUD / 地图也会更直接指向相位回投台，减少“载入基线后先做什么”的分叉。
- 真实 GUI 复测已确认回传第一版到当前段落无明显阻断；近期主线可从“继续观察回传是否成立”转入“实现回传上线后的下一包深段内容”。
- 恢复后的前哨核心现已承担基地整备点：回基地可直接恢复生命与防护，避免修复凝胶和旧图材料耗尽后形成软锁；左侧调试面板也新增 GM 资源 / 状态调整区，方便开发复测补状态。
- 回传上线后的下一包深段内容已继续推进到相位井锁：玩家从基地相位回投台重返前线后，会先沿“裂相碎屑 -> 透镜胚片 -> 中继调谐镜 -> 裂相尖塔”带回第一份内层故障轨迹，再回基地把它解析成相位井坐标印片。
- 内层故障轨迹后的后续链也已接入：玩家会返回裂相尖塔更东侧击退内层潜猎体、回收故障残渣，回处理点污染过滤器稳定出故障芯，再回基地基础反应器组装相位井钥，最后返回相位井锁带回第一份相位井定位器。
- 相位井定位器后的内层相位井包已落地：玩家现在会回基地解析定位器整理出相位井路由片，再沿新井口路线击退相位井哨戒体、回收井涌碎屑，回处理点污染过滤器稳定出相位井稳流芯，回基地基础反应器组装相位井探针，最后返回更东侧内层相位井带回第一份相位井芯样本。
- 相位井芯样本后的井底裂口包已落地：玩家现在会回基地解析井芯样本整理出相位井频谱片，再沿更东侧井底裂口路线击退井底潜伏体、回收井壁余烬，回处理点污染过滤器稳定出相位井稳相格，回基地基础反应器组装井底穿钉，最后返回井底裂口带回第一份相位井心核。
- 这轮继续复用污染过滤器和基础反应器，没有引入第三台设备、新地图切换或额外复杂系统；任务链、HUD 提示、地图目标、交互 prompt、旧存档补激活、保存校验、开发基线和 GM 资源候选项都已同步到新链路。
- 当前原型仍是工程演示，不代表正式画面、正式战斗、正式存档 UI 或完整首小时内容。

## 当前最高优先级

1. 快速推进一个可基本游玩的 demo：优先扩内容、场景、地图范围和新的基地 -> 外勤闭环，不再把近期主线压在护栏完善上；当前默认先用 `S7` / `S8` 观察“相位井定位器 -> 相位井路由片 -> 井涌碎屑 -> 相位井稳流芯 -> 相位井探针 -> 内层相位井 -> 相位井芯样本 -> 相位井频谱片 -> 井壁余烬 -> 相位井稳相格 -> 井底穿钉 -> 井底裂口 -> 相位井心核”是否顺滑。
2. 保护「基地服务冒险，冒险反哺基地」的核心闭环；每一包新内容都应让基地加工、补给、模块或通行能力真实改变下一次外勤结果。
3. 任务链数据维护、验证脚本职责、正式 HUD / 调试 HUD 边界和配方选择体验改为被动维护项；只在新内容开发暴露真实同步成本、回归风险或严重摩擦时补强。
4. 人工复测改为分层执行：默认先用开发基线定位段落，再按需补 `S0` 全链路空档复测。当前深段内容开发优先使用 `S7` / `S8`，需要回看相位井锁前置时再退回 `S6`，需要回看回投节奏时再退回 `S5`；若只差关键资源或生命 / 防护，直接使用调试 HUD 的 GM 区，而不是回头重打整条链。
5. 保持入口文档简约；阶段历史和长清单记录到周志或专题文档。

## 当前不做

- 不把联机、官方账号、交易、公会、排行榜、云存档或复杂服务端作为首版前置条件。
- 不把完整 Web 客户端、Android 或官方工具作为 MVP 必须交付项。
- 不把 Godot 客户端核心逻辑押到 C# / .NET。
- 不大规模生产美术资产，不提前批量编写玩家 Wiki 百科条目。
- 不在首小时同时解锁完整管线、多设备自动化、多天气、多角色 Build、大型任务日志、复杂技能树或联机入口。
- 不主动扩展存档边界校验、电网、管线、自动物流和多设备调度，除非真实改动需要。
- 不把参考仓库或历史旧仓库代码整包迁入本仓库。

正式规范见 `docs/architecture/code-style-and-language-practices.md`。当前阶段尤其需要避免为了拆分 HUD 或验证脚本而制造晦涩封装；新抽象必须能降低复杂度、减少真实重复或隔离清晰边界。

## 当前默认验证

客户端相关改动优先执行聚合验证：

```powershell
pwsh ./scripts/check-client.ps1
```

该脚本按顺序执行：

```powershell
pwsh ./scripts/check-client-data.ps1
pwsh ./scripts/check-client-scenes.ps1
pwsh ./scripts/check-client-save.ps1
pwsh ./scripts/check-client-quests.ps1
pwsh ./scripts/check-client-flow.ps1
pwsh ./scripts/check-godot-client.ps1
```

其中 `check-client-data.ps1` 已覆盖静态数据引用、本地化键、配方解锁来源一致性、任务激活重复声明、区域任务索引、主线任务图一致性和任务目标来源，避免 `unlock_conditions`、`next_quest_ids`、任务 `unlock_effects`、`quest_refs`、主线前后置和目标可获得性脱节；`check-client-scenes.ps1` 已覆盖第一切片任务目标与地图交互实例、建造点、加工设备和敌人实例的对应关系，并会按场景真实区域落点校验任务目标与 `quest_refs` 的区域归属同步。

提交前仍需执行仓库文本卫生和 diff 检查：

```powershell
pwsh ./scripts/check-text-files.ps1
git diff --check
```

脚本通过不等于首小时体验通过；文案理解、节奏感受、配方查找成本和重复跑图仍需空存档人工试玩判断。

## 阶段退出条件

- 玩家能在无开发者口头解释下完成至少一次「外出获得资源 -> 回基地加工 -> 远征产物改变下一次外勤结果」闭环。
- 玩家能理解基地后勤、设备缺料、地图目标、采集、基础战斗、制造远征产物、污染与防护、处理点建设、危险区域挑战和后续入口。
- 失败或被挡回时，UI、日志、地图标记或设备状态能给出具体补救方向。
- HUD、日志、方向提示、设备面板、小地图、任务完成反馈、撤离反馈和常驻状态摘要之间不遮挡、不信息过载。
- `check-client.ps1`、`check-text-files.ps1`、`git diff --check` 持续通过，阶段边界和协作规则同步更新到 `docs/`、`AGENTS.md` 和 `CLAUDE.md`。
