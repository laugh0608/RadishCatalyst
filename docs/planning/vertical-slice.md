# Vertical Slice

更新时间：2026-05-07

## 文档目的

这份文档用于定义 RadishCatalyst 在初始化 Godot 工程后的第一个可玩切片。

它不是完整 MVP，也不是正式 Demo。它只回答：

- 第一条可玩链路是什么。
- 做到什么程度算“方向成立”。
- 哪些系统必须接起来，哪些暂时用简化实现。
- 哪些内容不能在第一切片中扩张。

## 切片目标

第一可玩切片要验证：

**基地平台 -> 晶体矿脉区 -> 采集 / 战斗 / 采样 -> 回基地加工 -> 制造过滤模块 / 药剂 / 地基 -> 进入污染边界 -> 小型精英挑战 -> 解锁下一入口。**

玩家完成这条链路后，应明确感受到：

- 外出带回的资源不是库存数字，而是基地加工的输入。
- 基地加工产物能直接改变下一次远征能力。
- 污染边界不是单纯扣血地板，而是资源、危险和主线线索的入口。
- 装备、药剂、模块、地基和设备共同服务区域推进。

## 切片范围

### 场景范围

必须包含：

- 一个基地平台。
- 一个晶体矿脉区。
- 一个污染边界区。
- 一个锁住的后续入口，例如遗迹入口、污染深处或异常裂隙。

暂不包含：

- 完整星球。
- 多区域开放地图。
- 多星球。
- 大型随机地图。

### 玩家动作

必须包含：

- 移动。
- 基础攻击。
- 采集。
- 采样。
- 拾取。
- 使用快捷物品。
- 与基地设备交互。
- 建造或放置一个基础地板 / 地基。

可临时简化：

- 闪避。
- 技能。
- 多武器切换。
- 复杂采集工具差异。

### 基地系统

必须包含：

- 基础库存。
- 一个加工设备，例如反应器或过滤器。
- 一个储存设施。
- 一个基础地板 / 地基单元。
- 一个能制造远征产物的配方。
- 一种设备运行状态，例如工作中、缺料、完成或堵塞。

可临时简化：

- 完整电网。
- 完整管线网络。
- 多设备联动。
- 自动物流。

### 资源与配方

必须包含：

- 一种晶体或矿物资源。
- 一种污染沉积物或污染液资源。
- 一种样本。
- 一条加工链。
- 一个反哺远征的产物。

推荐链路：

```text
导电废件 + 晶体矿物 -> 反应器校准件
异常样本 + 异常残留物 -> 样本分析结果
晶体矿物 + 基础零件 -> 基础过滤模块
污染沉积物 + 溶剂 / 过滤处理 -> 抗性药剂或净化剂
异常样本 -> 解锁污染边界通行提示或后续入口
```

### 战斗与危险

必须包含：

- 一种基础敌人。
- 一种污染区敌人或受扰生态。
- 一个小型精英节点。
- 生命和防护压力。
- 污染对防护、敌人或建造的影响。

暂不包含：

- 复杂 Boss 阶段。
- 大规模敌潮。
- 大量敌人类型。
- 职业技能循环。

### 地形与建设

必须包含：

- 可破坏 / 不可破坏对象。
- 可建造 / 不可建造地块。
- 简单清理。
- 简单平整或基础地基。
- 建造失败原因提示。

暂不包含：

- 大规模地形雕刻。
- 多层建筑。
- 复杂承重。
- 真实土方模拟。

### 任务与叙事

必须包含：

- 一条主线任务链，串起整个切片。
- 两到三个支线或可选目标，用于教学和补资源。
- 一个结尾钩子，提示更危险区域或遗迹异常。

暂不包含：

- 大量对话。
- 多结局。
- 分支剧情。
- 完整角色关系网。

## 推荐任务链

1. 恢复前哨  
   玩家在基地平台恢复基础库存和加工设备。

2. 勘探晶体矿脉  
   玩家前往低危资源区，采集晶体矿物并遭遇基础敌人。

3. 采样并分析异常
   玩家采样异常晶体，就近回收残留物，再回基地加工分析。

4. 制造过滤模块  
   基地加工资源，制造过滤模块、抗性药剂或净化剂。

5. 扩建处理点  
   玩家清理地块，铺设基础地基，放置或启用污染处理设备。

6. 进入污染边界  
   玩家使用新产物进入更危险区域。

7. 治理污染源点  
   玩家战斗并处理污染节点，获得关键样本。

8. 解锁后续入口  
   地图打开遗迹入口或污染深处的提示，切片结束。

## 交付标准

第一切片完成时，应能连续游玩 15 到 30 分钟，并满足：

- 玩家无需开发者解释也能知道下一步目标。
- 每个关键交互都有可见反馈。
- 玩家能失败、撤离、补给并再次尝试。
- 存档能保存角色、库存、基地、地块、污染、任务和解锁状态。
- UI 能显示生命、防护、快捷栏、任务目标、库存、设备状态和建造失败原因。
- 至少一次“基地制造成果改变外勤结果”的体验成立。

## 首轮工程落地顺序

Godot 工程已初始化后，第一轮实现不直接追求完整切片，而是先建立可运行骨架：

1. `Boot` 进入 `GameRoot`，能加载一张临时切片地图。已完成。
2. `DataRegistry` 读取第一批静态数据，至少包含物品、配方、建筑、敌人、区域和任务。已完成。
3. `WorldState`、`CharacterState`、`InventoryState`、`QuestState` 能被创建、序列化和恢复。已完成默认创建、字典序列化入口和原型单文件存档保存 / 读取。
4. 玩家能在临时地图中移动，并触发一个采集点、一个敌人和一个基地设备交互。已完成玩家移动、采集 / 采样、前哨核心交互、基础攻击命中和基础反应器多配方轮换加工命令。
5. HUD 能显示当前目标、生命、防护、快捷栏和背包摘要。已完成当前目标、生命、防护、快捷栏、背包摘要和交互日志。

这轮完成后再进入采集、加工、建造、污染和存档的完整闭环。

- `Boot` / `GameRoot` 启动链。

- 前哨核心恢复后会保留可见并切换为已恢复颜色，提供第一步任务完成后的场景内反馈。

- `GatherSystem` 规则层雏形，交互节点不再直接发奖励，并已校验静态数据中的工具标签。

- 右侧状态面板已完成 UI 正式化后的第一轮信息量收敛，只保留目标、进度、生存状态、污染 / 模块、快捷栏和少量关键物资；区域方向、长提示、坐标和完整背包摘要已交给小地图、设备面板、交互提示、反馈面板或日志承担。
- `PrototypeHud` 已从多职责 HUD 主脚本收敛为控件胶水，主要负责节点引用、Label 赋值、面板显示 / 隐藏、计时和日志入口；日志文案规则已下沉到 `HudLogPresenter`，后续不应为了“拆分”继续过度抽象简单计时或控件操作。

仍未完成：

- 更完整的基地设备界面交互表现；当前只完成基础反应器和污染过滤器的雏形，不包含拖拽背包、自动物流、电网、管线或多设备管理。
- 更完整的小地图、正式地图标记表现和正式教程提示系统；当前只完成四个固定关键区域的状态标记，不包含缩放、寻路、动态地图或资产化图标，右侧状态面板只保留短摘要。
- 更完整的战斗失败 / 撤离节奏和正式死亡表现；当前只完成自动撤回前哨的结果面板雏形，不引入死亡惩罚或复活系统。
- 完整任务系统、教程提示和更正式的任务完成界面表现；当前只完成任务完成短时反馈雏形。
- 正式多槽位管理、设备运行时来源和更完整的跨块存档内容一致性复验；下一轮不主动扩展这部分，除非新增真实可保存状态或暴露实际坏档问题。

## 正式设备面板雏形

第一轮冷启动试玩已经证明任务链路可推进，设备面板雏形只解决“设备当前在做什么、为什么不能做、下一步该按什么”的理解断点，不先做大范围 UI 重构。

最小实现范围：

- 只覆盖基础反应器和污染过滤器。
- 复用现有设备交互入口，在玩家靠近设备时用 `Q` 打开 / 关闭设备详情面板；右侧日志和 HUD 状态继续保留为原型调试信息。
- 面板显示设备名称、当前运行状态、当前配方、可切换配方列表、输入 / 产出 / 副产、加工耗时、加工进度、最近完成结果、背包缺料项和下一步操作。
- 配方切换和启动加工继续走现有 `ProcessingSystem`、配方解锁、设备状态和缺料判断，不新增平行规则。
- 默认不新增存档字段；除非后续确认为必要，不保存面板打开状态或光标选择状态。

暂不纳入：

- 全局 HUD 重构。
- 拖拽式背包、仓库转移或批量投料。
- 电网、管线、自动物流和多设备调度。
- 正式小地图、任务完成界面、撤离 / 死亡表现和战斗节奏调整。

## 非目标

第一切片不追求：

- 画面最终品质。
- 完整数值平衡。
- 完整剧情表现。
- 全部装备系统。
- 完整自动化工厂。
- 完整天气系统。
- 联机。
- 大规模内容量。

## 风险控制

如果实现中遇到延期，优先保留：

1. 基地 -> 外出 -> 回基地 -> 再深入的闭环。
2. 存档和稳定 ID。
3. 任务目标和引导。
4. 采集、加工、制造、通行门槛。
5. 一种敌人和一种污染压力。

可以先砍：

- 第二种武器。
- 复杂天气。
- 多种地基。
- 多条支线。
- 额外敌人。
- 装饰性 UI。
- 继续加深存档坏档边界。
- 更多解释性 HUD 文本。

## 当前阶段结论

Godot 工程初始化后，不应先做“通用大框架”，而应围绕这条切片搭最小可运行结构。

只要第一切片能证明：

**外出获得资源和样本 -> 基地加工 -> 产出远征能力 -> 进入更危险区域**

项目就具备继续扩展的基础。

当前下一轮不再围绕小修和护栏收敛打转，而应直接推进下一包 demo 内容。任务链数据、目标来源、区域归属和场景实例已有检查可继续守住状态链路；后续只有在新内容开发暴露真实问题时，才按需补强这些护栏。

## 下一包 Demo 内容

建议把下一包内容定义为“遗迹外圈”第一版，用来验证第一切片之后的第二条可玩闭环，而不是只把现有地图继续拉长。

这包应包含：

- 在封锁遗迹入口之后补一段新的可走区域或更大的连续地图段。
- 至少加入 1 个新的外勤阻挡 / 危险，以及 1 个新的资源、调查点或推进目标。
- 优先复用基础反应器和污染过滤器，产出 1 个能明显改变下一次深入结果的开路物、环境对策、补给或通行能力。
- 形成“外出受阻 -> 回基地加工 -> 再次深入 -> 达成新目标”的第二闭环。

这包暂不做：

- 为此提前扩写新的通用大系统。
- 只放大地图面积但不增加新的风险、收益和基地回报。
- 大范围 UI 重构、正式教程框架、电网、管线、自动物流或第三台基地设备；除非内容验证已经明确被它们阻塞。

完成判定：

- 玩家能明确感受到基地准备改变了遗迹外圈的推进方式，而不是只重复第一切片已有动作。
- 新区域的阻挡、收益和回基地加工价值在无口头解释下仍可理解。
- 新内容若暴露任务链、HUD 或验证缺口，再按需补强，不把预支工程治理当成这包内容的前置。
