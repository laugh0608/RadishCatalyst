# Vertical Slice

更新时间：2026-04-28

## 文档目的

这份文档用于定义 RadishCatalyst 在初始化 Godot 工程后的第一个可玩切片。

它不是完整 MVP，也不是正式 Demo。它只回答：

- 第一条可玩链路是什么。
- 做到什么程度算“方向成立”。
- 哪些系统必须接起来，哪些暂时用简化实现。
- 哪些内容不能在第一切片中扩张。

## 切片目标

第一可玩切片要验证：

**基地平台 -> 晶体矿脉区 -> 采集 / 战斗 / 采样 -> 回基地加工 -> 制造过滤模块 / 药剂 / 地基 -> 进入污染边界 -> 小型精英挑战 -> 解锁下一入口。**

玩家完成这条链路后，应明确感受到：

- 外出带回的资源不是库存数字，而是基地加工的输入。
- 基地加工产物能直接改变下一次远征能力。
- 污染边界不是单纯扣血地板，而是资源、危险和主线线索的入口。
- 装备、药剂、模块、地基和设备共同服务区域推进。

## 切片范围

### 场景范围

必须包含：

- 一个基地平台。
- 一个晶体矿脉区。
- 一个污染边界区。
- 一个锁住的后续入口，例如遗迹入口、污染深处或异常裂隙。

暂不包含：

- 完整星球。
- 多区域开放地图。
- 多星球。
- 大型随机地图。

### 玩家动作

必须包含：

- 移动。
- 基础攻击。
- 采集。
- 采样。
- 拾取。
- 使用快捷物品。
- 与基地设备交互。
- 建造或放置一个基础地板 / 地基。

可临时简化：

- 闪避。
- 技能。
- 多武器切换。
- 复杂采集工具差异。

### 基地系统

必须包含：

- 基础库存。
- 一个加工设备，例如反应器或过滤器。
- 一个储存设施。
- 一个基础地板 / 地基单元。
- 一个能制造远征产物的配方。
- 一种设备运行状态，例如工作中、缺料、完成或堵塞。

可临时简化：

- 完整电网。
- 完整管线网络。
- 多设备联动。
- 自动物流。

### 资源与配方

必须包含：

- 一种晶体或矿物资源。
- 一种污染沉积物或污染液资源。
- 一种样本。
- 一条加工链。
- 一个反哺远征的产物。

推荐链路：

```text
晶体矿物 + 基础零件 -> 基础过滤模块
污染沉积物 + 溶剂 / 过滤处理 -> 抗性药剂或净化剂
异常样本 -> 解锁污染边界通行提示或后续入口
```

### 战斗与危险

必须包含：

- 一种基础敌人。
- 一种污染区敌人或受扰生态。
- 一个小型精英节点。
- 生命和防护压力。
- 污染对防护、敌人或建造的影响。

暂不包含：

- 复杂 Boss 阶段。
- 大规模敌潮。
- 大量敌人类型。
- 职业技能循环。

### 地形与建设

必须包含：

- 可破坏 / 不可破坏对象。
- 可建造 / 不可建造地块。
- 简单清理。
- 简单平整或基础地基。
- 建造失败原因提示。

暂不包含：

- 大规模地形雕刻。
- 多层建筑。
- 复杂承重。
- 真实土方模拟。

### 任务与叙事

必须包含：

- 一条主线任务链，串起整个切片。
- 两到三个支线或可选目标，用于教学和补资源。
- 一个结尾钩子，提示更危险区域或遗迹异常。

暂不包含：

- 大量对话。
- 多结局。
- 分支剧情。
- 完整角色关系网。

## 推荐任务链

1. 恢复前哨  
   玩家在基地平台恢复基础库存和加工设备。

2. 勘探晶体矿脉  
   玩家前往低危资源区，采集晶体矿物并遭遇基础敌人。

3. 带回样本  
   玩家采样异常晶体或污染残留，回基地分析。

4. 制造过滤模块  
   基地加工资源，制造过滤模块、抗性药剂或净化剂。

5. 扩建处理点  
   玩家清理地块，铺设基础地基，放置或启用污染处理设备。

6. 进入污染边界  
   玩家使用新产物进入更危险区域。

7. 治理污染源点  
   玩家战斗并处理污染节点，获得关键样本。

8. 解锁后续入口  
   地图打开遗迹入口或污染深处的提示，切片结束。

## 交付标准

第一切片完成时，应能连续游玩 15 到 30 分钟，并满足：

- 玩家无需开发者解释也能知道下一步目标。
- 每个关键交互都有可见反馈。
- 玩家能失败、撤离、补给并再次尝试。
- 存档能保存角色、库存、基地、地块、污染、任务和解锁状态。
- UI 能显示生命、防护、快捷栏、任务目标、库存、设备状态和建造失败原因。
- 至少一次“基地制造成果改变外勤结果”的体验成立。

## 首轮工程落地顺序

Godot 工程已初始化后，第一轮实现不直接追求完整切片，而是先建立可运行骨架：

1. `Boot` 进入 `GameRoot`，能加载一张临时切片地图。已完成。
2. `DataRegistry` 读取第一批静态数据，至少包含物品、配方、建筑、敌人、区域和任务。已完成。
3. `WorldState`、`CharacterState`、`InventoryState`、`QuestState` 能被创建、序列化和恢复。已完成默认创建和字典序列化入口，尚未接真实存档文件。
4. 玩家能在临时地图中移动，并触发一个采集点、一个敌人和一个基地设备交互。已完成玩家移动、采集 / 采样、前哨核心交互、基础攻击命中和基础反应器多配方轮换加工命令。
5. HUD 能显示当前目标、生命、防护、快捷栏和背包摘要。已完成当前目标、生命、防护、背包摘要和交互日志；快捷栏尚未完成。

这轮完成后再进入采集、加工、建造、污染和存档的完整闭环。

## 当前原型状态

截至 2026-04-28，临时原型已经具备：

- `Boot` / `GameRoot` 启动链。
- 首批 JSON 静态数据和 `DataRegistry`。
- `WorldState`、`CharacterState`、`InventoryState`、`QuestState` 默认状态。
- 一张 `VerticalSliceMap` 临时地图，包含基地平台、晶体区和污染区色块。
- 玩家移动、前哨核心交互、晶体采集、异常采样和污染沉积物采集。
- `GatherSystem` 规则层雏形，交互节点不再直接发奖励。
- 基础攻击输入、近距离敌人命中、敌人 HP 变化、击败反馈和敌人状态记录。
- 敌人受击后会进行即时反击，普通敌人造成生命压力，污染敌人额外造成会受过滤模块倍率影响的防护压力。
- `ProcessingSystem` 规则层雏形，基础反应器可在 `recipe.process_crystal_ore`、`recipe.make_filter_media`、`recipe.repair_gel` 和 `recipe.basic_filter_module` 之间轮换，并执行当前配方。
- 基础过滤模块最小启用命令，玩家可用 `F` 将背包中的 `equipment.filter_module_t1` 装入防护服模块槽，降低污染沉积斑交互造成的防护消耗。
- 污染边界区临时污染过滤器，可执行 `recipe.cleanse_residue`，将污染沉积物和基础溶剂处理为抗污染药剂并产出污染浆液副产物。
- 污染边界区基础敌人 `enemy.polluted_skitter`，拥有更高 HP、污染区颜色区分和击败反馈。
- 最小任务推进雏形，HUD 可显示当前目标进度，恢复前哨、采集晶体、采样异常晶体、制造过滤模块、启用模块、采集污染沉积物和击败污染敌人会推进对应目标。
- 可读性更高的原型 HUD。

仍未完成：

- 基地设备面板、加工耗时、配方解锁限制和设备运行状态表现。
- 更明确的区域通行门槛。
- 击败掉落反馈和更完整的战斗失败 / 撤离节奏。
- 任务目标进度统计。
- 工具标签校验、建造、污染防护消耗和真实存档文件。

## 非目标

第一切片不追求：

- 画面最终品质。
- 完整数值平衡。
- 完整剧情表现。
- 全部装备系统。
- 完整自动化工厂。
- 完整天气系统。
- 联机。
- 大规模内容量。

## 风险控制

如果实现中遇到延期，优先保留：

1. 基地 -> 外出 -> 回基地 -> 再深入的闭环。
2. 存档和稳定 ID。
3. 任务目标和引导。
4. 采集、加工、制造、通行门槛。
5. 一种敌人和一种污染压力。

可以先砍：

- 第二种武器。
- 复杂天气。
- 多种地基。
- 多条支线。
- 额外敌人。
- 装饰性 UI。

## 当前阶段结论

Godot 工程初始化后，不应先做“通用大框架”，而应围绕这条切片搭最小可运行结构。

只要第一切片能证明：

**外出获得资源和样本 -> 基地加工 -> 产出远征能力 -> 进入更危险区域**

项目就具备继续扩展的基础。
