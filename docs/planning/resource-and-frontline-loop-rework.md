# Resource And Frontline Loop Rework

更新时间：2026-05-16

## 背景

`S0` 新档冷启动人工复测已确认当前主线可以从 0 玩到终点，首小时引导可玩阶段可以收口。该文档记录的「资源循环与前线节奏重构」已于 2026-05-16 按阶段通过处理，后续只作为背景和回归口径。

- 基础零件长期依赖有限晶体簇，后段主线会因为早期采集物被耗尽而卡死。
- 深段后段任务重复度过高，连续出现“打怪 -> 采集 -> 回基地制造 -> 返回解锁”的同构流程，且往返基地成本过高。

本阶段目标不是继续扩线性内容长度，而是重构资源供给和前线节奏，让“基地服务冒险，冒险反哺基地”的闭环更稳定、更有变化。后续若只发现目标机械、空间拥挤或文案不顺等 `P2` / `P3` 问题，不再阻塞下一组核心循环开发。

## 设计原则

- 晶体簇只承担早期教学、地图探索奖励和应急补给，不再作为后段基础零件主供给。
- 基地应在前线推进后获得稳定补给能力，避免主线依赖有限世界物件。
- 回基地应代表整备、选择、升级、批量加工或解锁新远征能力，不应只是跑回去按一次制造。
- 前线目标应按区域机制区分，而不是用同一套门禁、敌人和采集模板换名字。
- 自动检查必须覆盖资源预算，避免资源断档只能靠人工长流程试玩发现。

## 基础零件供给方案

短期采用“三层供给”。

第一层：早期晶体簇仍保留。

- 用于教学玩家采集、加工和基础反应器输出。
- 用于给新档前期提供启动资金。
- 数量不再按后段全主线需求设计。

第二层：基地稳定补给成为主供给。

- 解锁前哨核心、污染过滤器或前线回传后，基地获得“晶簇碎片 / 前线副产物回收”能力。
- 最小实现可以是基地补给箱：每解锁一个大区域或完成一个关键阶段，补给箱生成一批 `基础零件` 或 `晶体矿物`。
- 文案口径是基地回收周边晶簇碎片、拆解旧设备残件或整理前线回传数据换来可用零件。
- 玩家缺料时，HUD 和设备面板应指向补给箱或前哨核心，而不是只提示“缺少晶体矿物”。
- 当前长期轻量产线落点是相位中继锚点部署后的 `回收基础零件`：基础反应炉可把 `污染浆液 x1` 低效率回收为 `基础零件 x2`，用于承接后段误操作缓冲，不引入电网、管线或新设备。

第三层：野外富晶残脉作为可选补给。

- 每个大区域可放 1-2 个富晶残脉或晶簇残脉。
- 它们用于奖励探索和兜底误操作，不作为主线必采目标。
- 地图提示只在玩家真的缺基础零件时引导最近补给来源，避免把扫矿变成隐性主线。
- 当前第一处落点是晶体矿脉区北侧 `富晶残脉`：它在首次晶体采集完成后才开放，缺基础零件时加工日志和设备面板会指向它，采集后仍需通过 `处理晶体矿物` 转成基础零件。

## 配方和消耗调整

- `处理晶体矿物` 当前已从 `基础零件 x2` 提高到 `x4`，这是临时缓冲，不是最终治理。
- `回收基础零件` 已接在相位中继锚点之后，只在玩家进入第二轮深段循环后把污染过滤副产物转回基础零件，避免抢走早期晶体采集教学。
- 后段配方应减少“基础零件固定税”：解析类配方已优先消耗阶段材料并移除固定基础零件输入，装配类配方仍可使用基础零件，但应优先被打包进整备任务或批量加工，不再单独形成一次返基往返。
- 每个阶段的基础零件净消耗应可被本阶段基地补给覆盖，并留出少量误加工余量。
- 新增配方时必须同时考虑资源预算检查，不允许只凭单次 GUI 流程判断够用。

## 资源预算检查

新增自动检查，目标是模拟主线最小通关路径和少量误操作：

- 从新档初始资源开始。
- 按主线必需配方扣除材料。
- 模拟每个大区域补给领取。
- 允许 2-3 次常见误加工或额外基础零件消耗。
- 断言不需要 GM 面板也能完成当前主线，并在终点保留合理基础零件缓冲。

该检查应作为 `check-client.ps1` 的一部分，至少在资源、配方、任务链或区域补给改动时覆盖。

## 前线节奏重构方案

从“前线回传锚点”之后开始重排深段后段节奏，不再连续使用同一套任务模板。

保留的核心闭环是：

```text
基地整备 -> 前线执行一组目标 -> 带回结果或建立回访收益 -> 基地升级下一次远征
```

回基地行为需要从“单件钥匙制造”改成以下类型之一：

- 远征整备：一次准备多个本阶段目标所需组件。
- 模块选择：选择偏向战斗、扫描、稳定、开路或回收的前线方案。
- 批量加工：把本阶段材料一次加工成后续连续目标所需的中间件。
- 回访升级：让已完成的前线点位变成回充、捷径、资源回收或重试点。

## 后段目标模板

下一阶段至少落地四类目标模板，避免连续三个任务同构。

现场校准：

- 玩家带着基地产物到现场校准节点。
- 目标重点是找对位置、按顺序校准或在干扰中维持进度。
- 可触发局部稳定窗口、地图显形或锚点升级。
- 当前第一版落点是 `校准稳窗相位序`：稳窗读数完成后，玩家返回井系桥东侧按西侧、中央、东侧顺序校准三处稳窗节点。

读数解锁采集：

- 玩家先处理现场读数点，降低材料节点的不稳定状态，再回收材料。
- 当前第一版落点是 `S9` / `S10` / `S12`：井心室分流读数、井纺室张力绕轮和井系桥结点分别打开心棘、纬束和系索回收线。
- `S5` 到 `S16` 首轮人工复测确认这些现场目标逻辑暂无明显问题；后续只在真实试玩中继续观察它们是否转成新的顺序打卡。

短守场：

- 已用于锚场回稳，可以继续作为少量高压节点。
- 失败后保留部署物并允许直接重试，不能退回到重采材料。
- 奖励应是回充点、捷径或区域稳定收益。
- 当前锚场版本已追加部署后的两处压力钉清理，守脉体在压力钉清完后才完全暴露，避免部署后只变成单怪检查。

路线清障：

- 玩家在多条路径中选择清障路线。
- 清障可以消耗不同模块，带来不同收益，例如更短回路、更多补给或更低敌压。
- 目标重点是路线选择，而不是固定打一个怪。
- 当前第一版落点是 `回收边缕残条`：井纹架边缘新增南北两条侧路障，玩家清理任一条后打开边缕残条回收线，另一条保留为未选路线。

局部定序 / 解谜：

- 用于相位井、井锁、导管束等区域。
- 玩家根据现场状态调整顺序、方向或匹配关系。
- 失败反馈应快速、低惩罚，不应导致长距离重跑。

## 跑图减压

- 每个大区域第一次进入可以保留完整路程，完成核心目标后应建立回投点、捷径或现场回稳收益。
- 前线回传锚点保持“多已部署落点 + 单当前回投目标”口径，用 `R` 切换已部署落点即可，不扩成自由传送网。
- 若任务要求回基地加工，应该尽量让玩家一次加工出下一趟外勤所需的一组物品；当前相位井后段 6 组“过滤中间件 + 装配件”已改为同一整备任务内完成。
- 可考虑在前线锚点附近提供临时装配能力，只处理当前阶段材料和任务件，避免回基地只做一个零件。

## 落地顺序

1. 实现基地补给箱或前哨核心回收，先解决基础零件稳定供给。
2. 加入资源预算自动检查，并把它接入客户端聚合验证。
3. 已审视前线回传锚点之后的配方，并把基础零件从后段解析类配方中移除。
4. 重排深段后段任务模板，优先改连续同构最严重的区域；已先收口前线回传后 9 处单件制造节点，并落地第一版现场校准和路线清障目标，`S5` 到 `S16` 首轮人工复测逻辑通过。
5. 阶段已通过；后续只在出现 `P0` / `P1` 或准备宣布大阶段通过时，再安排完整 `S0` 新档验证。

## 阶段退出标准

- 新档能从 0 完成当前主线，不需要 GM 面板补基础零件。
- 玩家缺基础零件时，有明确基地补给或地图补给方向。
- 后段至少出现四类目标模板，且不连续三个任务使用同一结构。
- 回基地至少在关键阶段承担整备、选择、升级或批量加工功能。
- 前线锚点、捷径或现场收益能显著减少重复长距离往返。
- `check-client.ps1`、`check-docs.ps1`、`check-text-files.ps1` 和 `git diff --check` 持续通过。
